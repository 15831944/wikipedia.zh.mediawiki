'''匿名方法''' (Anonymous Method) 是一種用於[[物件導向|物件導向]]程式設計的[[物件導向程式設計_方法|方法]]，此種程式撰寫的方式經常可以在[[事件驅動程式設計|事件驅動程式設計]]的領域中使用。
==簡介==
一般來說， 在[[物件導向程式設計|物件導向程式設計]]的領域中， 建立一個[[類別|類別]]， 我們通常會給定類別的方法，以提供必要的功能。

然而， 我們在定義一個[[類別方法|類別方法]]時， 通常要指定這個方法必要的資訊. 這個資訊包含了方法的名稱，呼叫後回傳值的型態，參數的數量以及型態. 這資訊我們通常稱之為[[方法原型|方法原型]](Method Prototype). 

然而，使用匿名方法的意思就是我們根本不用給定[[類別方法|類別方法]]的名稱。

因為使用這種方式定義的[[類別方法|類別方法]]，不必將程式碼分成兩部分: 定義方法部分與呼叫程式碼部分。

取而代之的是， 直接將[[類別方法|類別方法]]的定義寫在[[程式碼|程式碼]]會呼叫的地方. 當[[編譯器|編譯器]]看到這樣寫法[[程式碼|程式碼]]時， 會自動將作者寫的[[原始碼|原始碼]]展開成標準的格式， 然後才進行編譯.

使用匿名方法這種語法來撰寫[[程式|程式]]，可以讓[[程式碼|程式碼]]看起來更為簡潔。

一般來說，簡潔的程式碼通常比較容易維護。

==使用時機==
在[[事件驅動程式設計|事件驅動程式設計]] 的領域中，註冊一個[[物件導向程式設計_事件處理函式|事件處理函式]]，是程式設計師經常要做的事情。

然而，註冊一個[[物件導向程式設計_事件處理函式|事件處理函式]]，往往相當麻煩，而且該[[物件導向程式設計_事件處理函式|事件處理函式]]往往只在[[原始碼|原始碼]]中被[[程式設計_參考|參考]]一次。

針對這樣的情況，[[物件導向程式設計_事件處理函式|事件處理函式]]的名稱，就可以使用匿名方法的特性，進行簡潔化.

==支援的程式語言==
*[[C_Sharp|C#]] 2.0 版
*[[Java|Java 程式語言]]
*[[JavaScript|JavaScript]]
*[[PHP|PHP]]
*[[Python|Python]]

==程式範例==
下面是一個使用匿名方法的C#語言範例:

我們希望當 button1 這個按鈕，被[[滑鼠|滑鼠]]按下時，會呼叫這個[[函式|函式]]進行我們想要的處理

以前的方式 (要兩個步驟)
<pre>
 Step 1:事件處理函式
   void Button1Handler(object s，EventArgs e) {
        button1 的對應處理 ... 
   }

 Step 2: 將 Button1Handler 這個方法，向按鈕 button1 註冊
    button1.Click   +=   new    EventHandler( Button1Handler);
</pre>
-------------------------------------------------------------------------

'''使用匿名方法來實作上面的程式功能''':
<pre>
  button1.Click   +=   delegate(object    s,EventArgs   e) {
     button1 的對應處理 .... 
  };
</pre>
