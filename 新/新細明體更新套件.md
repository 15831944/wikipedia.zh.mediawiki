'''新細明體更新套件'''是[[微軟|微軟]]公司為[[Windows_XP|Windows XP]]及[[Windows_Server_2003|Windows Server 2003]]的中文版用戶，可以使用[[中日韓統一表意文字|表意文字補充平面]]的字元而製作的更新套件。它將系統字型[[新細明體|新細明體]]更新為Unicode 3.1的標準。由於補充平面上的字元使用了32位元的編碼空間，所以更新只適用於Windows XP或以上的作業系統。

==更新細明體==

===字數===
更新套件主要把原來的[[細明體|細明體]]和新細明體的字型檔(MingLiU.ttc)更新成新版：隨Windows XP送來的應該是3.21版，而更新後的版本號碼將會是5.03。這些在安裝了TrueType字型延伸屬性插件之後就可以看得到。新版本除了原來Unicode在基礎平面的20,902個Unicode 2.0的基本字元以外，還包括有6582個屬於擴展A區的字元。此外，其他屬於[[中日韓統一表意文字|擴展B區]]的字元，都另外放在一個叫MingLiUB.ttc的字，包含有42,711個漢字。兩個字型檔合共為系統提供總數70,195個漢字字型。

===字型外觀===
{{see also|國字標準字體}}
[[File:Ming_updated_SVG.svg|thumb]]
安裝了更新套件後，新細明體字型全面更改了外觀，由原本採用接近傳統[[明体|明体]]的[[傳承字形|字形]]，變成依從用了[[国字标准字体|国字标准字体]]。

修改前細明體採用「[[傳承字形|傳承字形]]」，亦稱作「[[康熙字典|康熙字典]]體」或「[[舊字形|舊字形]]」，即以傳統明體的字形爲宗，字的寫法較接近篆形，較符合字理。例如「禮」字的「⺬」旁，首兩筆是橫，符合「祭祀的桌子」之象形；「-{爲}-」字以手牽象，會意有作為，上方是「爪」部首，「爪」代表手；「讀」字右旁的聲符是「𧶠」，而不是完全不近音的「-{賣}-」；「請」的右方是「-{靑}-」，下作「円」，即「丹」字，符合「丹青」之義。

修改後的文字，則符合Unicode標準，以及符合[[中華民國教育部|中華民國教育部]]公佈的[[國字標準字體|國字標準字體]]。例如「-{為}-」這字元在Unicode裏是9筆的寫法（見圖），但舊版的細明體在某些文字大小時卻造成「-{爲}-」的字型，這個爪字頭的字在Unicode定義在不同的內碼。有執着於Unicode標準的用戶視舊版為「錯誤造字」，認為更新後的細明體「改正」了字形。還有偏旁「-{⺬／礻}-」、「-{青／靑}-」等，更新版本也改從國字標準字體的寫法，向手寫靠攏。

然而，也有意見指出明體的印版古籍中只有爪字頭的「-{爲}-」而沒有「-{為}-」，「-{為}-」字常見於古代手寫書法作品中，而非明體的印版裏，認為以「-{為}-」取代「-{爲}-」值得商榷；另外「系」字在新細明體中的改變，第一筆的撇與第二筆的折沒有相連，以傳統字書的角度來說也屬錯字。更新版細明體於字形正確方面都遭到詬病<ref name=wrong>參見[http://www.ideographer.com/articles/article.php?aid=67 符號工作站：《是誰寫錯字？》]系列、[http://founder.acgvlyric.org/iu/doku.php/%E8%AA%AA%E6%96%87:%E5%AE%8B%E9%AB%94%E6%BB%BE%E8%9B%8B_%E9%82%84%E6%88%91%E7%B4%B0%E6%98%8E 刻石錄：《宋體滾蛋，還我細明》]</ref>。

此外，由於國字標準字體的字樣，破壞了許多傳統明體的美學設計<ref name=wrong />，也可能因為不合印刷體設計原則或未針對螢幕顯示最佳化，所以常有使用者認為修改後的字比較難看。字型製作公司文鼎論及華航「帝雉號」採用更新後的細明體而惹來批評之事，就直斥微軟系統字新細明體改成教育部標準宋體寫法後，「已失掉日本寫研本蘭明朝體的印刷字體之美，成為失去平衡的手寫筆法字體，印刷界相當排斥」。<ref>[http://www.inside.com.tw/2016/10/27/china-airlines-font Inside：《華航「帝雉號」字體風波，文鼎：票選將影響品牌識別》]</ref>。而其董事長楊淑慧於更早時就已說明國字標準字體為何不適合印刷：「如果單純用手寫的筆法來定義印刷字體該長什麼樣子，字就會東倒西歪，閱讀感受會很差。」例如「肉字旁部首」的國字標準字體寫法，會影響視覺的延伸性。因此，像明體、黑體等印刷字型，並不遵循國字標準字體。她更批評局方閉門造車，指出當初制訂國字標準字體時，只是找一群訓詁學者閉門商討，並沒有邀請書法家和印刷業者等相關領域的專家一起討論，令討論出來的結果完全沒考慮到印刷的可行性與閱讀的便利性。<ref>[http://www.bnext.com.tw/article/view/id/37342 數位時代：《金萱的聲音我們聽到了！其他本土字型業者呢？文鼎：我們從未缺席！》]</ref>另一家字型製作公司Justfont也批評：「三十年前教育部的『台標』制定，不管是印刷廠或是書法家，全部人都驚呆了。這個標準沒有和民間溝通，都造成大家很大困擾。」並指出：「依照台灣標準的新細明體，細看會發現字的平衡和美感被破壞。」<ref>[http://www.biosmonthly.com/collection_topic/6553 BIOS monthly：《走過字體寒冬的春暖花開——專訪金萱字型開發justfont團隊》]</ref>

新細明體更新後，除了棄用[[舊字形|舊字形]]，設計上亦有多番視覺缺陷、平衡問題等毛病，一直備受漢字設計和排印界批評<ref>[https://thestandnews.com/personal/新細明體-早該被時代革除-卻又意外留下的審美遺毒/ 許潮文：《新細明體：早該被時代革除、卻又意外留下的審美遺毒》（立場新聞）]、[https://thestandnews.com/art/為甚麼不推薦新細明體/ 許潮文：《為甚麼不推薦新細明體》（立場新聞）]</ref>，甚至被要求改回 Windows XP 預載較美觀之版本<ref>[http://polymerhk.com/articles/2015/10/13/22745/ 鄭惟：《將 Windows 7 / 8 / Vista 醜陋的新細明體改回 Windows XP 預載較美觀之版本》（聚言時報）]</ref>。

===輸入法===
更新套件亦同時把系統原來原始只能輸入 Unicode 2.0 漢字的傳統注音輸入法和傳統倉頡輸入法，擴充到支援輸入擴充A區的漢字。不過，由於相容性的問題，要輸入擴充B區內的漢字則必須使用新版的微軟新注音、及新倉頡輸入法 6.0 版。（需額外下載，詳見[http://www.microsoft.com/taiwan/download/CHTdownload.htm 微軟公司下載網站]。）

==问题==
在发布后不久，该字体下载便被[[微软|微软]]自行移除。据称移除该字体下载的原因之一是许多人认为字体变丑了<ref>{{cite web |url = http://smileyskeleton.blogspot.com/2009/03/m.html | title=M$產品別亂裝之「新細明體更新套件」篇 | accessdate=2012-04-12}}</ref>，另一个原因则是安装后无法彻底卸载（被修改的字体文件不能自动还原）。因此，在安装前请对原文件（MingLiU.ttc）做好备份。<ref>{{cite web| language=| publisher= |title=[注意]微軟新細明體更新套件1.0版下載服務已被微軟自行移除!!!| url = http://forum.cpatch.org/archiver/?tid-6316.html| author=| date=| accessdate=}}{{dead link|date=2018年3月 |bot=InternetArchiveBot |fix-attempted=yes }}</ref>

=== 解決方案 ===
先前提到此更新套件是針對 Windows XP/2003 的作業系統所做的系統自行修正，使用者可從 Windows 7 字形資料夾內取得 7.0 版本的新細明體字形。

微軟公司後來推出了可在Windows XP及Windows Server 2003執行的ISO 10646:2003支援套件<ref>{{cite web | language = | publisher= |title = 支援 ISO 10646:2003 Windows XP 字形集 及 繁體中文支援修正1 |url = http://www.microsoft.com/downloads/zh-hk/details.aspx?FamilyID=1112aa2c-e011-471c-a12c-656e767d3bb8| author= | date= | accessdate= }}</ref>。该套件包括了6.90版本的新細明體，細明體，HKSCS細明體，細明體-ExtB，HKSCS細明體-ExtB，和新細明體-ExtB字形，但不包括輸入法的更新。

然而，面對字型設計醜陋的批評，微軟至今尚未有正式回應或行動。
