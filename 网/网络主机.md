'''网络主机'''（'''network host'''）是已连接到一个[[计算机网络|计算机网络]]的一台[[電子計算機|電子計算機]]或其他设备。网络主机可以向网络上的用户或其他[[节点_(电信网络)|节点]]提供信息资源、服务和应用。一台网络主机是已被分配一个[[网络层|网络层]]{{tsl|en|Host address|主机地址}}的网络节点。

使用[[TCP/IP协议族|TCP/IP协议族]]参与网络的计算机也可称为IP主机。更具体来说，参与[[互联网|互联网]]计算机可称为互联网主机，有时还称作互联网节点。互联网主机和其他IP主机在其网络接口上被分配一个或多个[[IP地址|IP地址]]。该地址可以由管理员手动配置，也可以在启动时自动由[[动态主机设置协议|动态主机设置协议]]（DHCP）或通过{{tsl|en|Stateless address autoconfiguration|无状态地址自动配置}}方法自动配置。

每台网络主机都是一个物理[[节点_(电信网络)|网络节点]]（即一个网络设备）{{fact}}，但不是所有物理网络节点都是一台主机。[[调制解调器|调制解调器]]、[[集線器|集線器]]或[[網路交換器|網路交換器]]等网络设备有可能不被分配{{tsl|en|Host address|主机地址}}（有管理目的除外），因此不被认为是网络主机。网络打印机、[[路由器|硬件路由器]]等设备有IP地址，但由于它们不是通用目的计算机，它们有时候不被认为是主机。

== 服务器与主机 ==
所有[[服务器|服务器]]都是主机，但并非所有主机都是服务器。任何与网络建立连接的设备都可以作为主机，而以接受其他设备（客户端）连接为目的的主机被视为服务器。

== 概念起源 ==
在[[操作系统|操作系统]]领域，术语[[终端主机|终端主机]]传统上是为[[終端|计算机终端]]提供服务的[[多用户|多用户]]计算机或软件，或一台提供较小或较少能力的计算机设备<ref>E. Garrison Walters, [https://books.google.com/books?id=AwrQsOW5SsQC&pg=PA149&dq=terminal+%22host+%22+a+multi-user&hl=sv&ei=1jwCTfudOcbysgb2xuCLCg&sa=X&oi=book_result&ct=result&resnum=1&ved=0CCkQ6AEwAA#v=onepage&q=multi-user%20host&f=falseThe essential guide to computing], page 149, Prentice Hall PTR, 2001.</ref>。例如一台[[大型计算机|大型计算机]]为[[电传|电传]]终端或[[終端|视频终端]]提供服务。其他例子是[[telnet|telnet]]主机（一个telnet服务器）和{{tsl|en|xhost}}（X窗口客户端）。

术语“互联网主机”或“主机”在定义互联网及其前身[[ARPANET|ARPANET]]的许多[[RFC|RFC]]文档中被使用.在开发ARPANET时，连接到网络的计算机一般是可以通过[[串行端口|串行端口]]从已连接终端访问的大型计算机。由于这些[[終端|虚拟终端]]本身没有主机软件或自己执行计算，它们不被视为主机。终端通过串行接口和可能通过电路交换网络连接到终端主机，但并未连接到任何基于IP的网络，并且未被分配IP地址。而现今的IP主机，可能无法作为终端主机{{what}}。

RFC 871将一台主机定义为：“连接到一个通信网络的通用计算机系统，用于在参与的操作系统之间实现资源共享目的”。<ref name="rfc871">RFC 871, ''A Perspective on the ARPANET Reference Model'', M.A. Padlipsky (September 1982)</ref>
