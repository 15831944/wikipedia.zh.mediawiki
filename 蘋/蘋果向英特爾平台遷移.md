'''蘋果向英特爾平台遷移'''指的是2006年[[麥金塔電腦|Mac電腦]]從[[PowerPC|PowerPC]]平台遷移至[[Intel|Intel]] [[x86|x86平台]]的事件。平台遷移計畫最初在2005年的[[WWDC|WWDC大會]]上由[[蘋果公司|Apple]]執行長[[史提夫·喬布斯|史提夫·喬布斯]]（Steve Jobs）正式公布，Apple稱日後將停止在Mac電腦中繼續使用由[[摩托羅拉|摩托羅拉]]（Motorola）及[[IBM|IBM]]提供的[[PowerPC|PowerPC]][[微處理器|微處理器]]，改用Apple大多數競爭對手選用Intel提供的x86[[處理器|處理器]]。<ref>Apple press release, June 6, 2005:[http://www.apple.com/pr/library/2005/jun/06intel.html ''Apple to Use Intel Microprocessors Beginning in 2006'']</ref>

這一次平台遷移已是Apple第三次更改旗下電腦產品線的[[CPU|CPU]]架構。Apple將[[Apple_II|Apple II]]的8位元[[MOS_6502|6502處理器]]改用成摩托羅拉的[[摩托羅拉_68000|68k處理器]]架構（首次配備在[[Macintosh_128k|Macintosh 128k]]上）是為第一次平台遷移；後來放棄使用摩托羅拉68k處理器，改用PowerPC處理器，則是為第二次平台遷移。迄今Apple仍然是業界裡唯一一家成功完成了核心部件平台遷移的企業（不考慮微軟[[Surface_2|Surface 2]]更新至[[Surface_3|Surface 3]]時從[[ARM架構|ARM架構]]回歸[[x86架構|x86架構]]），歷史上其他嘗試過類似平台遷移的企業最終均以失敗告終，如[[Commodore|Commodore]]、[[雅達利|雅達利]]（Atari）、[[Acorn_Computers|Acorn Computers]]、[[迪吉多|迪吉多]]（Digital）、[[SGI|SGI]]、[[Kaypro|Kaypro]]等<ref>[http://www.pcmag.com/article2/0,2817,933453,00.asp ''Apple Switch'']</ref><ref>[http://lowendmac.com/stotler/07/0104.html ''The Past and Possible Future of Apple's Intel Transition'']</ref>

Apple的最初發佈的新聞稿預計平台遷移將會在2006年6月開始行動，之後在2007年末結束，而實際上Apple的平台遷移進行得比料想中快得多。首代配備了Intel處理器的Mac電腦於2006年1月就正式推出，史提夫·喬布斯在2006年8月推出配備Intel處理器的[[Mac_Pro|Mac Pro]]時表示，硬件的平台遷移已進入最後階段：Apple將會在2006年10月推出最後一款需要遷移至Intel平台的產品─[[Xserve|Xserve]]。<ref>[http://macworld.com/news/2006/08/07/liveupdate/ "WWDC Live Keynote Update"] {{webarchive|url=https://web.archive.org/web/20070614170257/http://www.macworld.com/news/2006/08/07/liveupdate/ |date=2007-06-14 }}</ref> 最終新版Xserve伺服器於2006年12月推出，[[硬體|硬體]]層面的Intel平台遷移此時宣告完結。<ref>{{Cite web|url=http://images.apple.com/xserve/pdf/Xserve_TechnologyOverview12202006.pdf |title=Xserve Technology Overview|format=PDF}}</ref>

Apple於2009年8月28日推出了僅支援Intel處理器的[[Mac_OS_X_Snow_Leopard|Mac OS X 10.6 Snow Leopard]]，正式在新版作業系統中去除對PowerPC架構Mac電腦的支援，但仍然留有[[Rosetta|Rosetta]]等執行[[PowerPC|PowerPC]]應用程式所需的系統元件<ref name="applepr">{{cite press release | url=http://www.apple.com/pr/library/2008/06/09snowleopard.html | title=Apple Previews Mac OS X Snow Leopard to Developers | publisher=[[Apple_Inc.|Apple Inc.]] | date=2008-06-09 | accessdate=2008-06-11 }}</ref>。2011年推出的[[Mac_OS_X_Lion|Mac OS X 10.7 Lion]]則不再支援Rosetta。

儘管能夠運作在x86處理器上，但Apple官方目前仍然不支援在x86個人電腦上安裝[[macOS|macOS]]。在除了[[Intel_Mac|Intel Mac]]以外的x86相容電腦運行Apple軟體，見[[OSx86|OSx86]]。<ref>{{cite web|title=Patched Kernels|url=http://wiki.osx86project.org/wiki/index.php/Patched_Kernels|website=osx86project.org}}</ref>

== 往例 ==

在1994年到1996年期間，Mac電腦產品線同樣完成了一次類似的硬件平台遷移，此時Apple選擇了停止繼續使用摩托羅拉推出的68K處理器，改用由[[AIM聯盟|AIM聯盟]]（[[蘋果公司|Apple]]、[[IBM|IBM]]、[[摩托羅拉|摩托羅拉]]）共同開發推出的PowerPC處理器。這一次的平台遷移花費了數年時間，期間Apple推出了可以同時運行在68K處理器及PowerPC處理器的[[Mac_OS|Mac OS]][[作業系統|作業系統]]，並為配備了PowerPC處理器的Mac電腦製作了底層的[[模擬器|模擬器]]，鼓勵第三方開發商製作能夠原生運作在兩個平台的程式（因為該種程式程式碼肥大，又被用戶暱稱為「fat binaries」）。

稍後幾年，Apple更是將全線Mac電腦的作業系統，自原有的Mac OS遷移至全新的Mac OS X之中。該次平台遷移同樣花費了數年時間（小部分舊款Mac電腦依然使用早期的Mac OS作業系統），Apple其後在作業系統中集成的[[Classic環境|Classic環境]]使用戶可以方便地遷移至新的作業系統環境。Classic環境允許用戶在Mac OS X內刷出一個[[Mac_OS_9|Mac OS 9]]副本（Instance），在Mac OS X上執行一些未經移植的Mac OS 9程式。Apple同時還為Mac OS 9及Mac OS X推出[[Carbon_(API)|Carbon API]]，讓開發者可以更簡易地為兩個平台開發程式。

事實上，Apple為每代Mac OS X同時開發了Intel版本及PowerPC版本，而且還保留了對上代[[NeXTSTEP|NeXTSTEP]]作業系統的兼容性。Apple並沒有透漏過曾為哪些硬件平台開發過作業系統，而近年推出的[[iOS|iOS行動作業系統]]，修改自原有的OS X計畫，則顯而易見地是運作在[[ARM架構|ARM平台]]之上。

== 遷移原因 ==
史提夫·喬布斯表示，Apple遷移電腦產品硬件平台的主要原因是對IBM在PowerPC技術的開發進展上感到失望，同時相信Intel可以在日後可以更好地滿足Apple的需要與部署。在Intel提供的發展路線圖當中，史提夫·喬布斯曾特別指出，如果按照消耗的能源來作對比基準，以瓦特來作單位，Intel處理器的表現要遠遠佳於PowerPC處理器的性能。<ref name=autogenerated1>[http://www.macworld.com/article/46961/2005/09/intelvsamd.html Analysis: Why Apple picked Intel over AMD | Mac IT | Macworld<!-- Bot generated title -->]</ref> 而在手提電腦設計裡，這是一個非常重要的考慮因素，這一性能指標將嚴重影響電腦的電池壽命與使用時間。

2003年6月，史提夫·喬布斯在推出配備[[PowerPC_G5|PowerPC G5]]處理器的Mac時曾許諾，在十二個月內會將PowerPC處理器的[[時鐘頻率|時鐘頻率]]提昇至高達3 GHz。而實際上即便是在兩年之後，Apple與IBM依然未能將3 GHz的PowerPC處理器投入市場，市場上的流言認為這一尷尬的局面是IBM其[[POWER4|POWER4]]衍生芯片組的[[良品率|低良品率]]所造成的。此外，配備在Mac手提電腦上的PowerPC處理器，在運作時產生的巨大熱量，也廣被同時期的業界詬病。<ref>[http://forums.macrumors.com/archive/index.php/t-156771.html powerbook gets hot fast [Archive] - MacRumors Forums<!-- Bot generated title -->]</ref>

一些分析師曾預料Apple會與另一家x86處理器生產商[[AMD|AMD]]展開合作。然而與Intel相比，AMD缺少優良的低能耗CPU設計，相信這是Apple放棄與AMD合作的最終原因。<ref name=autogenerated1 />不過自2017年[[AMD_Ryzen|AMD Ryzen]]推出市場並且其效能、能耗表現良好（與英特爾[[Haswell微架構|Haswell]]/Boardwell微架構的處理器基本持平），而AMD有為客戶半客製晶片的業務對Apple而言又有成本優勢，加上英特爾自2012年以來的處理器產品成本高企而效能和能耗上進步又不高，因此有消息指Apple未來可能會考慮採用基於[[AMD_Zen|AMD Zen微架構]]的[[SoC|SoC]]產品。<ref>{{cite web|title=AMD’s Zen Processor Might Power Next-Generation Apple MacBook Pro – Custom SOC Design With HBM2 Memory System|url=http://wccftech.com/amd-zen-soc-apple-macbook-pro/|website=wccftech.com}}</ref>

== 優點 ==

最初，如要在Mac上執行x86軟體，需要類似如[[DOSBox|DOSBox]]或[[Microsoft_Virtual_PC|Microsoft Virtual PC]]這一類的虛擬化軟體。在Apple的電腦產品遷移至Intel平台之後，這一類軟體（如[[Parallels_Desktop_for_Mac|Parallels Desktop for Mac]]及[[VMware_Fusion|VMware Fusion]]）如今將可以透過虛擬化技術，提供更接近原生速度的虛擬環境。希望達致更方便的部署環境的用戶，透過在配備x86處理器的Mac上使用[[Boot_Camp|Boot Camp]]軟體，甚至可以劃分兩個、三個、甚至是四個啟動磁區，來原生啟動各種[[作業系統|作業系統]]。

雖然大多數遊戲使用[[Mac_OS_X|Mac OS X]]無法利用的[[DirectX|DirectX]] [[API|API]]，但遷移至Intel平台之後，開發者明顯可以更方便地製作出獨立的系統代碼，如透過使用[[OpenGL|OpenGL]]。至少開發者無需再處理移植至PowerPC平台時遇到的端序問題（Endian）與[[指令集|指令集]]依賴問題。

== 通用二進制 ==
{{Main|通用二進制}}
為了減輕開發人員移植平台時的負擔，Apple推出了[[通用二進制|通用二進制]]程式代碼，使應用程式能以原生形式運行在使用PowerPC或者Intel微處理器（x86）的Mac電腦上，在同一個程序包中同時為兩種架構提供最理想的性能。通用二進制與Apple的Intel平台遷移計畫同時在2005年WWDC大會上公布，開發者在轉用Intel平台的Mac電腦後，可以使用通用二進制來發佈同時適合新舊Mac電腦使用的應用程式。當程式在Mac OS X中執行後，作業系統將自動檢測通用二進制程式碼，然後根據使用的硬體架構自動選擇合適的程式碼來執行，因此實現原生的本地程式運行速度。因為同時包含了兩套架構的程式碼，通用二進制應用程式的大小要比普通版本的大一些。

[[Xcode|Xcode]]是唯一一個可以編譯通用二進制代碼的GUI工具，在Xcode 2.4之中，Apple進一步擴充了通用二進制程式碼的功能，引入「四架構二進制」概念（對應Intel及PowerPC平台的32位元及64位元版本），使支援64位元運算的PowerPC G5處理器在Apple正式遷移至Intel平台後的歲月裡依然可以發揮餘熱。

隨着去除PowerPC支援的[[Mac_OS_X_Snow_Leopard|Mac OS X Snow Leopard]]的發佈，部分應用程式已經去除了對PowerPC處理器的支援。

== 遷移的障礙 ==
=== 公眾對平台遷移的反應 ===

Apple宣布遷移至Intel平台後推出了過渡方案[[Rosetta|Rosetta]]─一套PowerPC的動態轉譯器，用戶可以通過使用Rosetta在配備Intel處理器的Mac上執行PowerPC應用程式。<ref>[http://www.apple.com/hk/rosetta Apple - Rosetta<!-- Bot generated title -->]</ref> 然而，在Rosetta最初公布的時候，其實際效能僅有G3處理器的60%～80%。除此之外，在PowerPC Mac中，為虛擬Mac OS 9環境而設的Classic環境，在遷移至x86平台之後也將不復存在，使得新的配備了Intel處理器的Mac電腦無法執行最原始的Mac OS Classic應用程式（然而，用戶可以使用由第三方開發者推出的虛擬軟件，如[[SheepShaver|SheepShaver]]，來虛擬一台執行Mac OS 9的Mac電腦）。
也有用戶質疑，Intel處理器甚至是[[x86|x86]]架構其本身的性能是否足夠強大，平台遷移會否影響到作業系統的性能與應用程式品質。除此之外用戶關注的問題還包括：Intel平台上的記憶體端序順序，以及應用程式實作時，相比同等級別或同時代PowerPC處理器的[[浮點|浮點]]運算性能下降。

在Apple正式推出配備Intel處理器的Mac電腦之前，用戶社群曾有不少疑惑（如擔心過早公布遷移事項會造成恐慌效應，導致產品滯銷；或是擔心主導[[USB|USB標準]]的Intel會要求Apple停止在新型Mac電腦上使用Apple自行研發的[[FireWire|FireWire硬體介面]]），但最終均被證明全是莫需有的擔心，Apple最終很好地處理了平台遷移事務上的過渡。

Apple在[[Macworld|Macworld]] 2006正式發佈首款配備Intel處理器的Mac電腦，用戶心中的疑惑大多都隨著正式產品的推出而煙消雲散<ref>Anandtech, January 30, 2006: [http://www.anandtech.com/mac/showdoc.aspx?i=2685 Apple Makes the Switch: iMac G5 vs. iMac Core Duo]</ref>。Rosetta在正式面世時得到了大幅改進，實際的執行效率比最初演示的速度要快得多。（當然，在Intel Core Duo iMac之下，使用Rosetta執行PowerPC程式碼的速度依然比不上[[G5_iMac|G5 iMac]]的速度。）Intel的Core Duo處理器其實際性能，在Mac OS X上的表現幾乎可以與原先的Power Mac G5桌上型電腦媲美。Intel處理器在[[整數|整數]]運算及[[浮點|浮點]]運算[[SPECmark|SPECmark]]測試之中的表現，甚至還超越了2005年的G5 iMac型號。

可以同時在Intel Mac和PowerPC Mac運作的應用程式（如[[Safari|Safari網頁瀏覽器]]），在Intel平台上的代碼其執行效率要優於在PowerPC平台上的性能。然而，與PowerPC Mac相比，配備了Intel處理器的新型Mac電腦卻無法直接執行Classic環境的應用程式。如果需要在Intel Mac上執行Mac OS X推出之前的Classic應用程式，則只能使用第三方開發者推出的虛擬工具，如vMac、Basilisk II、[[SheepShaver|SheepShaver]]等，這些工具或多或少都受到硬件或軟件架構上的制限，功能及穩定性不足。

公眾之前對平台遷移計畫的疑惑在Apple實際推出產品之後旋即煙消雲散，財報顯示在2006年的聖誕購物季裡面，Apple的Mac機銷量比上一年同期大幅增長。而「Intel Inside」這一類通常在Intel OEM客戶產品上會出現的標示，也從沒出現在任何Apple產品之上。

=== 硬體 ===

外界一直不了解在整個平台遷移過程之中，Apple保留了多少非處理元件的系統設計權重。Apple傳統上是一個整合系統的建造商，一些評論擔心Apple的[[工業設計|工業設計]]理念會受到其他通用部件的設計方式而影響。對此持不同意見的人則指出，Apple事實上正逐步使用與[[PC|PC]]一致的通用部件（如於1995年為[[Power_Mac|Power Mac]]添加的[[PCI|PCI介面插座]]<ref>[http://lowendmac.com/history/1995dk.shtml 1995: Clones, the Worst Macs, Pippin, PCI Slots, and CPU Daughter Cards]</ref>），轉用非Apple設計的晶片組與產品設計，並未影響到Mac的實際表現效能與形象。

配備Intel處理器的Mac在產品[[軔體|軔體]]上選用了全新的Intel技術─[[可延伸韌體介面|可延伸韌體介面]]（EFI）─而非以往Apple一直使用的[[Open_Firmware|Open Firmware]]技術。採用EFI技術的電腦，無需再倚賴普通PC上的[[BIOS|BIOS元件]]，就可以同時提供到更多的可延伸性功能。<ref>[http://developer.apple.com/documentation/MacOSX/Conceptual/universal_binary/universal_binary_diffs/chapter_3_section_10.html Apple's transition documentation: Extensible Firmware Interface]</ref>

遷移至[[x86|x86]]架構也使得[[Windows|Windows]]可以原生運行在Apple硬體上，開發者也可以嘗試使用[[Wine|Wine]]的組件來直接在Mac OS X上執行Windows應用程式。有人認為，[[Mac_OS_X|Mac OS X]]用戶自此可以在配備了Intel處理器的Mac上安裝多一個Windows作業系統，或使用Wine（或其變體[[CrossOver|CrossOver Mac]]或Darwine）執行Windows應用程式，會使得Mac OS X在軟體開發的布局裡變得沒那麼重要。反對者則表示，遷移至Intel平台對[[Mac_OS_X|Mac OS X]]來說利大於弊：希望轉用Mac電腦的用戶現在購買Mac之後，不必離開現有的PC作業系統，即可一睹Mac OS X的芳容。也有用戶提出，Apple的電腦在遷移至Intel平台之後是否可以算作是一台普通PC的討論，Apple對此則表示，不會允許普通PC執行Mac OS X<ref>[http://images.apple.com/legal/sla/docs/macosx106.pdf Mac OS X Snow Leopard 軟體許可協議]</ref>，因此遷移至Intel平台後的Mac依然有其獨特性，不能算作是一台普通PC。對此限制，民間則建立了一些社群，嘗試在普通PC上安裝並執行Mac OS X，該計畫被稱為[[OSx86|OSx86]]計畫或Hackintosh（中文社群時常將其稱為「[[黑蘋果|黑蘋果]]」<ref name = bmr>{{cite web | url = http://benchmarkreviews.com/index.php?option=com_content&task=view&id=623&Itemid=38 | title = Turning PC into Apple Macintosh: Hackintosh | accessdate = 2010-10-10 | author = David Ramsey  | publisher = BenchmarkReviews.com}}</ref>）。

=== Boot Camp ===
{{Main|Boot Camp}}

在Apple推出[[Boot_Camp|Boot Camp]]軟體前，用戶與開發者普遍認為，因[[Windows_XP|Windows XP]]並不支援EFI軔體架構，因而無法在新款的Intel Mac上使用<ref>[[Engadget|Engadget]], January 11, 2006: [http://engadget.com/2006/01/11/no-xp-on-intel-macs-but-vista-is-good-to-go/ New Macs not XP friendly, but Vista is good to go]</ref><ref name=onmac>Onmac.net, March 16, 2006: [http://onmac.net/ Windows XP on an Intel Mac]</ref>，微軟在回答用戶疑問時則表示，32位元版本的[[Windows_Vista|Windows Vista]]並不支援EFI軔體架構，但可以透過安裝最新版的Boot Camp軟體，讓Windows Vista運行在Intel Mac之上。

在以往的Mac社群當中，Intel處理器往往被視為又熱又慢的產品（尤其是[[奔騰4|奔騰4]]）。在1990年代末，Apple甚至還拍攝過一部名為「烤兔子」<ref>[http://www.youtube.com/watch?v=e6PoLiXCA40 Power Mac G3 Toasted Bunnies]</ref> 的廣告來諷刺奔騰II處理器的發熱問題與性能。然而，專為手提電腦設計的[[奔騰M|奔騰M]]系列處理器，運作時的溫度則一直要比奔騰4處理器涼快得多。Apple聲稱，Intel推出的新款[[Intel_Core|Intel Core]]處理器，乃是基於原有的奔騰M微架構，若以能源消耗作對比基準，以[[瓦特|瓦特]]來作基本對照單位，Intel新款處理器的表現要比[[PowerPC_G5|PowerPC G5]]甚至是[[Power_PC_G4|G4]]都要好得多。

對是次平台遷移抱負面態度的人士認為，[[x86|x86]]並非一個良好的選擇，較PowerPC相比，Intel的x86處理器缺少[[硬體暫存器|硬體暫存器]]，且也不支援PowerPC的[[AltiVec|AltiVec技術]]。支持者則聲稱，x86架構自[[Intel_8086|8086處理器]]最初推出時已經過大量改進與發展，CPU內部的設計隨著時間的推進，也開始融合起[[精簡指令集|精簡指令集]]及[[複雜指令集|複雜指令集]]其雙方的設計理念，與以往的處理器設計早已完全不一樣。他們還認為，[[SSE|SSE指令集]]在未來的改進，最終有可能發展得比AltiVec技術更為優越。

Intel Core Solo與Intel Core Duo處理器均屬[[32位元|32位元]]設計。2006年8月7日，Apple推出首款採用[[Xeon|Xeon]]處理器的[[64位元|64位元]]架構[[Mac_Pro|Mac Pro]]及[[Xserve|Xserve]]產品。到了2007年8月7日，Apple旗下所有電腦產品線均已更新至支援64位元運算的[[Intel_Core_2_Duo|Intel Core 2 Duo]]處理器。

按當時的性能基準來比較，Intel Core 2 Duo處理器的性能在32位元運算之下，並沒有比Intel Core Duo處理器要優秀很多，其64位元架構的性能優勢，直至[[Mac_OS_X_Snow_Leopard|Mac OS X 10.6 Snow Leopard]]面世才真正得以發揮出來。在配備了64位元Intel處理器的Mac電腦裡面，用戶也可以安裝[[Windows_7|Windows 7]]使用完整的64位元模式來啟動電腦。

=== 現有的PowerPC應用軟件 ===
{{Main|Rosetta}}

運行在直譯器上的[[Java|Java]]應用程式（不依賴於Java原生介面的）、[[Dashboard|Dashboard]] Widget、及工序指令無需改動即可同時運作在兩種系統上<ref>[http://www.netneurotic.net/mac/intel/index.html PowerPC/Intel Mac執行應用程式對比]</ref>。無法遷移至Intel平台的Mac OS X應用程式，其代碼若需要在Intel Mac上使用，將會自動傳送至PowerPC動態轉譯器「[[Rosetta|Rosetta]]」中執行。[[Rosetta|Rosetta]]最初僅支援[[PowerPC_G3|G3]]的指令集，但後來經Apple改進，目前支援[[AltiVec|AltiVec技術]]以及[[PowerPC_G4|G4]]的指令集，僅餘下[[PowerPC_G5|G5]]處理器的附加指令集並未提供支援<ref>{{cite web
 |url         = http://developer.apple.com/legacy/mac/library/documentation/MacOSX/Conceptual/universal_binary/universal_binary.pdf
 |title       = What Can Be Translated?
 |work        = Universal Binary Programming Guidelines, Second Edition
 |publisher   = Apple
 |accessdate  = 2007-07-04
 |deadurl     = yes
 |archiveurl  = https://web.archive.org/web/20120327121744/http://developer.apple.com/legacy/mac/library/documentation/MacOSX/Conceptual/universal_binary/universal_binary.pdf
 |archivedate = 2012-03-27
}}</ref>，因此必須基於G5處理器的應用程序無法使用在Rosetta上執行。即便已是相當老舊的程式版本，絕大多數使用G4指令集以下的應用程式都可以無誤地直接在Rosetta上正常運作，如於2003年推出的[[Internet_Explorer_for_Mac|Internet Explorer for Mac 5.2.3]]。[[Rosetta|Rosetta]]是一個類似先前允許PowerPC Mac電腦在68k模擬器中執行舊程式的指令轉譯器。與Classic環境不同的是，[[Rosetta|Rosetta]]並不需要在啟動程式前預先開啟一個子系統即可馬上工作。
[[File:Intel_Mac_with_PowerPC_application.png|thumb]]正在執行PowerPC應用程式[[Internet_Explorer_for_Mac|Internet Explorer for Mac]]]]

[[AltiVec|AltiVec技術]]自[[Mac_OS_X_v10.3|Mac OS Panther]]起本身已被封裝成一個向量程式庫，該向量程式庫在[[PowerPC|PowerPC]]平台上會調用[[AltiVec|AltiVec技術]]，在[[x86|x86]]平台上則使用[[SSE|SSE指令集]]。

新版[[Xcode|Xcode]]已支援同時為Intel及PowerPC平台的Mac電腦編譯[[通用二進制|通用二進制]]代碼。[[Cocoa|Cocoa]]應用程式如需製作通用二進制版本代碼，只需要檢查一下端序問題，並重新編譯一次即可完成移植。Carbon應用程式移植時可能需要一些額外的調整，但其複雜程度遠比從Mac OS 9過渡至Mac OS X時低。使用[[Metrowers|Metrowers]] [[CodeWarrior|CodeWarrior]]套件編寫的應用程式則可能需要大幅修改，Apple與Metrowerks稱使用[[PowerPlant|PowerPlant]]的應用程式稍作修改即可完成移植。

採用x86架構的Mac電腦將不再支援Classic環境。配備了Intel處理器的[[麥金塔電腦|Mac]]將無法再直接執行Mac OS Classic環境的軟體。然而，配備了Intel處理器的Mac電腦，可以選擇安裝Mini vMac、Basilisk II及[[SheepShaver|SheepShaver]]來虛擬一個[[Mac_OS|Mac OS]] Classic環境的[[作業系統|作業系統]]，以此來執行某些舊式程式。
