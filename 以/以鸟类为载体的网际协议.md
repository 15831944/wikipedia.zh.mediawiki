在[[计算机网络|计算机网络]]技术中，'''以鸟类为载体的网际协议（IP over Avian Carriers, IPoAC）'''是一个理论上使用[[信鸽|信鸽]]等[[鸟类|鸟类]][[传输|传输]][[网际协议|IP]]数据的通讯提案。

IPoAC最初由[[IETF|互联网国际标准机构]]在'''[[:rfc:1149|RFC 1149]]'''中提出（1990年4月1日，[[惡搞RFC|愚人節RFC]]），作者为D. Waitzman。尽管互联网国际标准机构经常在4月1日发布一些恶搞之作，他们还是让这个提案看上去像是一个真的、非虚构的提案。其後，互联网国际标准机构於1999年[[愚人節|愚人節]]發表了'''[[:rfc:2549|RFC 2549]]'''，為IPoAC「加上」[[QoS|QoS]]功能，作者同樣是D. Waitzman。

在2011年愚人節，互联网国际标准机构再發表了《RFC 1149在IPv6的應用》（'''[[:rfc:6214|RFC6214]]'''）。

IPoAC曾经被成功实验，但是9个封包中只有4个封包成功传送，丢包率55%（由用户失误造成），响应时间从54分钟到1.77小时不等<ref>{{cite web |url = http://www.blug.linux.no/rfc1149/ |title = RFC-1149 |work = BLUG |deadurl = yes |archiveurl = https://web.archive.org/web/20111004221306/http://www.blug.linux.no/rfc1149/ |archivedate = 2011-10-04 }}</ref>。因此，该技术的延迟问题严重。但是，该技术可以由数据封包本身增大而得到大幅改进，如使用信鸽携带大容量[[存储卡|存储卡]]。

== 风险 ==
2005 年 12 月，[[高德纳咨询公司|高德纳公司]]在一篇关于禽流感的报告中给出的结论 “大型传染病不会直接对信息系统造成影响” ，因为忽视了 RFC 1149 和 RFC 2549 而受到了幽默的批评。<ref>{{cite web|url=http://articles.yuikee.com.hk/newsletter/2005/12/g.html|title=RFC1149 and RFC2549 的病毒风险|publisher=|language=en}}</ref>

== 其他信鸽数据传输应用 ==
户外[[摄影师|摄影师]]经常使用信鸽作为传输数码照片的工具。信鸽可以用一小时跨越约30[[英里|英里]]距离传输数十[[GB|GB]]的数据，速度比一般的[[ADSL|ADSL]]快很多。即使将存储卡丢失因素也考虑进去，依然不失为一个值得考虑的数据传输方法。<ref>[http://www.notes.co.il/benbasat/5240.asp "Israeli pigeons faster than ADSL"] {{webarchive|url=https://web.archive.org/web/20080713090722/http://www.notes.co.il/benbasat/5240.asp |date=2008-07-13 }}</ref>

这种简单的大容量传输并不使用[[IP协议|IP协议]]。
