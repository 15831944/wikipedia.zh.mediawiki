== 结构及命名 ==
在设计上，瑞晶使用一种“模块化”的方法，允许它加载与攻击目标相定制的功能，从而实现特定的间谍活动，使得瑞晶非常适合针对目标的持久、长期、大规模的监视<ref name=":0" /><ref>{{Cite web|url=http://hackingpost.com/regin-malware-state-sponsored-spying-tool-targeted-govts/|title=Regin Malware - 'State-Sponsored' Spying Tool Targeted Govts|work=The Hacking Post - Latest hacking News & Security Updates|access-date=2019-01-18|archive-url=https://web.archive.org/web/20170218064037/http://hackingpost.com/regin-malware-state-sponsored-spying-tool-targeted-govts/|archive-date=2017-02-18|dead-url=no}}</ref><ref name="scmagazine">{{Cite web|url=http://www.scmagazineuk.com/nsa-gchq-or-both-behind-stuxnet-like-regin-malware/article/384888/|title=NSA, GCHQ or both behind Stuxnet-like Regin malware?|accessdate=2014-11-25|date=2014-11-24|publisher=scmagazineuk.com|archive-url=https://web.archive.org/web/20160616183435/http://www.scmagazineuk.com/nsa-gchq-or-both-behind-stuxnet-like-regin-malware/article/384888/|archive-date=2016-06-16|dead-url=no}}</ref>。这与[[火焰_(恶意软件)|火焰]]等其他恶意软件的方法相同，有些功能还与2011年9月发现的{{Le|Duqu|Duqu}}恶意软件类似<ref name=":0"/>。其背后的黑客可以通过监视屏幕、远程控制等渠道来定制攻击的方法<ref name=":1">{{Cite news|url=https://www.cr173.com/html/47041_1.html|title=Regin是什么 Regin病毒软件有什么危害|author=|date=2014-11-25|work=西西软件园|accessdate=|archive-url=https://web.archive.org/web/20150512032232/http://www.cr173.com/html/47041_1.html|archive-date=2015-05-12|dead-url=no}}</ref>。瑞晶还可以捕获和传输密码、恢复已经删除的文件、监控网络流量<ref name=":1" />。 

但瑞晶也是隐秘的，不会在受感染的系统上存储多个文件；相反，它经过多层加密，使用自己的加密[[虛擬檔案系統|虚拟文件系统]]，它包含在一个从名称上来看无害的单个文件中，并采用了很少使用的[[RC5|RC5密码]]的变体加密<ref name="scmagazine"/>。 

赛门铁克表示，这款恶意程序有五个阶段，它只在执行完第一阶段后才会执行下一阶段，每一个阶段“都非常隐蔽和加密，除了第一阶段”。还称瑞晶的每一阶段所能提供的有关完整程序包的信息都非常有限，只有拦截了全部五个阶段，才有可能分析和理解具体的威胁。<ref name=":0"/><ref name=":1"/>

[[赛门铁克|赛门铁克]]表示，它和[[卡巴斯基實驗室|卡巴斯基]]都将恶意软件定义为Backdoor.Regin。Backdoor即中文中的“[[后门|后门]]”。<ref name=":2" /><ref name="symantecblog">{{Cite web|url=http://www.symantec.com/connect/blogs/regin-top-tier-espionage-tool-enables-stealthy-surveillance|title=Regin: Top-tier espionage tool enables stealthy surveillance|accessdate=2014-11-25|date=2014-11-23|publisher=Symantec|archive-url=https://web.archive.org/web/20181126193746/https://www.symantec.com/connect/blogs/regin-top-tier-espionage-tool-enables-stealthy-surveillance|archive-date=2018-11-26|dead-url=no}}</ref>卡巴斯基和赛门铁克都发布了[[白皮书|白皮书，]]其中包含了解恶意软件的相关信息。 <ref name="whitepapers">{{Cite web|url=http://www.symantec.com/content/en/us/enterprise/media/security_response/whitepapers/regin-analysis.pdf|title=Regin White Paper|accessdate=2014-11-23|publisher=Symantec}}</ref><ref name="whitepaperk">{{Cite web|url=https://securelist.com/files/2014/11/Kaspersky_Lab_whitepaper_Regin_platform_eng.pdf|title=Regin White Paper|accessdate=2014-11-24|publisher=Kaspersky Lab|archive-url=https://web.archive.org/web/20141127064402/https://securelist.com/files/2014/11/Kaspersky_Lab_whitepaper_Regin_platform_eng.pdf|archive-date=2014-11-27|dead-url=yes}}</ref> 

== 发起者 ==
{{See also|{{le|全球监管披露 (2013年至今)|Global_surveillance_disclosures_(2013–present)}}}}
德国新闻杂志《[[明镜|明镜]]》在2013年6月报道称，[[美国国家安全局|美国国家安全局]]对[[欧洲联盟|欧盟]]的公民和机构进行了在线监控。这些信息来自前国家安全局工作人员[[爱德华·斯诺登|爱德华·斯诺登]]获得的秘密文件，他在2013年6月揭露了[[美国|美国]]从2007年开始的[[稜鏡計畫|棱镜计划]]，并完整曝光了与瑞晶相关的US-984XN监控计划。该计划年度预算为2000万美金，可监控从手机通讯到网络通讯的各种通讯方式，且由美国国家安全局直接介入。<ref name=":2" /> 

{{Le|The Intercept|The Intercept}}报道称，2013年时[[政府通信总部|英国政府通信总部]]袭击了[[比利时|比利时]]最大的电信公司[[Belgacom|Belgacom]]<ref name="intercept20041124"/>。这些攻击可能导致瑞晶引起安全公司的注意。根据IT安全公司Fox IT的分析，Regin是英国和美国情报机构的工具。Fox IT在其客户的计算机上找到了Regin，根据他们的分析，Regin的部分在{{Le|NSA ANT目录|NSA ANT_catalog}}提及，名称为“Straitbizarre”和“Unitedrake”。《明镜》则称Fox IT的客户是Belgacom。<ref name="nsa-und-gchq">{{cite web|url=http://www.spiegel.de/netzwelt/netzpolitik/trojaner-regin-ist-ein-werkzeug-von-nsa-und-gchq-a-1004950.html|title=Spionage-Software: Super-Trojaner Regin ist eine NSA-Geheimwaffe|first1=Christian|last1=Stöcker|first2=Marcel|last2=Rosenbach|date=2014-11-25|publisher=|via=Spiegel Online|access-date=2019-01-18|archive-url=https://web.archive.org/web/20181111093242/http://www.spiegel.de/netzwelt/netzpolitik/trojaner-regin-ist-ein-werkzeug-von-nsa-und-gchq-a-1004950.html|archive-date=2018-11-11|dead-url=no}}</ref> 

== 影响 ==
在瑞晶全球感染的计算机中，28％在[[俄罗斯|俄罗斯]]，24％在[[沙特阿拉伯|沙特阿拉伯]]，9％在[[墨西哥|墨西哥]]和[[爱尔兰岛|爱尔兰]]，5％在[[印度|印度]]、[[阿富汗|阿富汗]]、[[伊朗|伊朗]]、[[比利时|比利时]]、[[奥地利|奥地利]]和[[巴基斯坦|巴基斯坦]]<ref name="symantecblog"/>。

过去六年里，Regin已在对世界多个目标进行攻击。目前被发现的来自瑞晶的攻击，近半都是针对互联网服务供应商、电信运营商等价值高的企业的顾客。瑞晶的攻击领域还包括能源、航空、研究部门、医院等。但攻击范围并不仅限于这些高价值的目标，近半感染设备来自小企业和普通民众<ref name=":1"/>。

2014年12月，德国报纸《[[图片报|图片报]]》报道，瑞晶在[[安格拉·默克尔|安格拉·默克尔]]的一名工作人员使用的[[闪存盘|USB闪存驱动器]]上被发现。[[德国总理府|德国总理府]]的所有高安全性笔记本电脑的检查显示没有其他感染。<ref>{{Cite news|url=http://www.dw.de/german-government-denies-falling-victim-to-cyber-attack/a-18158951|title=German government denies falling victim to cyber attack|date=2014-12-29|work=Deutsche Welle|access-date=2019-01-18|archive-url=https://web.archive.org/web/20150506200808/http://www.dw.de/german-government-denies-falling-victim-to-cyber-attack/a-18158951|archive-date=2015-05-06|dead-url=no}}</ref>
