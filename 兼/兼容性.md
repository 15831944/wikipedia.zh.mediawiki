'''兼容性'''（{{Lang-en|compatibility}}），是電腦軟體、遊戲軟體、作業系統、系統工具等一切與[[電腦|電腦]]有關的作業程序應用時所可能遇到的一種情形。

== 兼容性類別 ==
# [[互操作性|相互性]]（{{lang-en|Interchangeability}}），與不同電腦、同款軟體進行應用時幾乎無任何錯誤。如[[Microsoft_Office|Microsoft Office]] 、[[Nero_Burning_ROM|Nero Burning ROM]]、[[IOS|IOS]]。
# [[向上兼容|舊系統相容性]]（{{lang-en|Forward compatibility}}），即新系統程式可支援舊系統程序。但可能產生崩潰或未知的問題。許多時候需要配合相容性設定使用。如[[Microsoft_Office|Microsoft Office]]、[[AutoCAD|AutoCAD]]、[[Windows_Media_Player|Windows Media Player]]。
# [[向下兼容|新系統相容性]]（{{lang-en|Backward compatibility}}）係指從舊系統考量到支援與否，如[[windows_98|windows 98]]、[[windows_95|windows 95]]、[[windows_2000|windows 2000]]、[[windows_me|windows me]]、甚或[[windows_xp|windows xp]]等許多系統軟體功能不適用在新系統上。必要時採取放棄手段停止支援。 <ref>[http://e-words.jp/w/%E5%BE%8C%E6%96%B9%E4%BA%92%E6%8F%9B.html 後方互換とは｜後方互換性｜バックワードコンパチビリティ｜backward compatible - 意味/定義 ： IT用語辞典]</ref>
# 軟體版本相容性，在軟體更新上常會有更新部分功能表或程序碼的情形。例[[Safari|Safari]] 5.1.7支援[[windows|windows]]，但其後版本停止支援。分成上位相容性和下位相容性兩種：上位相容性（{{lang-en|Upper compatibility}}），如2.0的軟體版本可以支援1.0的系統程序；下位相容性（{{lang-en|Lower compatibility}}），如1.0的軟體版本功能被2.0包含。

== 兼容性的軟體 ==
根據[[EULA|EULA]]，[[LGPL|LGPL]]，[[多許可|多許可]]的情況下，開始發展[[自由軟體|自由軟體]]。最著名的如[[Open_Office|Open Office]]、[[Libreoffice|Libreoffice]]、[[WPS_Office|WPS Office]]等軟體可以完美的支援[[WORD|WORD]]檔案的編輯撰寫。
這種情形與市場需求有關聯之外，也可能在不同系統程序上編寫。實現了真正的互換相容性。有些專家認為這種情性卻有助於原始的程式軟體更多人使用。

又如[[Google_Chrome|Google Chrome]]、[[Internet_Explorer|Internet Explorer]]、[[FireFox|FireFox]]排版瀏覽器已多授權[[原始碼|原始碼]]提供創作者開發瀏覽器。故也產生在[[linux|linux]]除了firefox也能使用IE排版瀏覽器，或[[IOS|IOS]]上可使用Google排版瀏覽器等情形。

== 優缺點 ==
廣義而言，已存在的系統和軟體並沒有必要再去提供支援。

有些廠商可能會堅持在某系統上才能使用該軟體，又或者是不願支援某些特定系統。

如[[主機板|主機板]]若不支援[[windows_10|windows 10]]的最低要求，則安裝後會產生無法支援該系統而崩潰或停擺的情形。

== 詬病 ==
[[IE_Only|IE Only]]是一[[微軟公司|微軟公司]]獨特給予[[Internet_Explorer|Internet Explorer]]上瀏覽部分網頁的功能，例如[[合作金庫|合作金庫]]網頁進行線上修改密碼時，只能透過IE。此功能用意是強迫用戶不得透過火狐與谷歌進行操作。故在台灣許多公務機關與公司行號仍使用XP電腦，仍內建著IE瀏覽器。呈現一種時代潮流在進步，電腦科技卻停滯的特殊情況。<ref>[https://tw.news.yahoo.com/不想讓chrome-黨獨大-是時候給firefox機會了-063100151.html 不想讓Chrome「一黨獨大」？是時候給Firefox機會了！]</ref>
