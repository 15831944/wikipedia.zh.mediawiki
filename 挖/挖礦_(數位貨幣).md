'''挖礦'''（{{Lang-en|Mining}}），是獲取[[比特幣|比特幣]]的勘探方式的暱稱。由於其工作原理與開採[[礦物|礦物]]十分相似，因而得名。此外，進行挖礦工作的比特幣勘探者也被稱為'''礦工'''。

== 工作原理 ==
'''比特幣礦工'''通過解決具有一定工作量的[[工作量證明機制|工作量證明機制]]問題，來管理比特幣網路 —— 確認交易並且防止[[雙重支付|雙重支付]]。

[[中本聪|中本聰]]在他的論文中闡述說：

{{cquote|在沒有中央權威存在的條件下，既鼓勵礦工支援比特幣網路，又讓比特幣的貨幣流通體系也有了最初的貨幣注入源頭。}}

中本聰把通過消耗CPU的電力和時間來產生比特幣，比喻成金礦消耗資源將黃金注入經濟。
比特幣的挖礦與節點軟體主要是透過[[對等網路|對等網路]]、[[數位簽章|數位簽章]]、[[互動式證明系統|互動式證明系統]]來進行發起零知識證明與驗證交易。

每一個網路節點向網路進行[[廣播|廣播]][[交易|交易]]，這些廣播出來的交易在經過礦工（在網路上的電腦）驗證後，礦工可使用自己的[[工作量證明機制|工作證明]]結果來表達確認，確認後的交易會被打包到資料塊中，資料塊會串起來形成連續的資料塊鏈。中本聰本人設計了第一版的比特幣挖礦程式，<ref name="processors">{{Cite news|url=http://www.wired.com/magazine/2011/11/mf_bitcoin/all|title=The Crypto-Currency|last=Davis|first=Joshua|date=10 November 2011|newspaper=[[Wired_Magazine|Wired Magazine]]|accessdate=11 November 2011}}</ref>這一程式隨後被開發為廣泛使用的第一代挖礦軟體[[Bitcoin|Bitcoin]]，這一代軟體從2009年到2010年中旬都比較流行。

每一個比特幣的節點都會收集所有尚未確認的交易，並將其歸集到一個資料塊中，礦工節點會附加一個隨機調整數，並計算前一個資料塊的[[SHA-256|SHA-256]][[散列函數|雜湊運算]]值。挖礦節點不斷重複進行嘗試，直到它找到的隨機調整數使得產生的雜湊值低於某個特定的目標。

由於[[雜湊演算法|雜湊運算]]是不可逆的，尋找到符合要求的隨機調整數非常困難，需要一個可以預計總次數的[[暴力破解|不斷試錯]]過程。這時，工作量證明機制就發揮作用了。當一個節點找到了符合要求的解，那麼它就可以向全網廣播自己的結果。其他[[節點|節點]]就可以接收這個新解出來的資料塊，並檢驗其是否符合規則。如果其他節點通過計算雜湊值發現確實滿足要求（比特幣要求的運算目標），那麼該資料塊有效，其他的節點就會接受該資料塊。

== 挖礦礦產 ==
除了將接收到的交易資訊打包到資料塊，每個資料塊都會允許「發行」一定數量的新比特幣，用來激勵成功發現資料塊的礦工。

比特幣系統按照預定的貨幣增發節奏決定發行的比特幣數量。如果其他支付交易有給手續費的，那麼礦工還會獲得手續費。由於礦工可以自行決定是否將某一個交易資料打包到資料塊中，因此礦工有可能優先選擇手續費較高的交易來打包。

資料塊產生速率的預期為每10分鐘一個，但每個資料塊中，新發行的比特幣不能超過50個，而這個數字每產出21萬個區塊就會減半，大約每4年就會發生一次。

因此，比特幣的總數量不會超過2100萬個。隨著新發行比特幣數量的下降，手續費將成為挖礦的主要動機。

那些在較早時期將電腦資源投入到比特幣挖礦活動中的用戶，相對於較晚加入的礦工而言，獲得比特幣容易得多。這樣設計的原因，主要是為了保證在比特幣的早期發展階段可以吸引足夠的計算力來處理資料塊。事實上，如果沒有人挖礦，比特幣的初期交易活動就無法處理，比特幣經濟也將停止運行。<ref>[http://www.donews.com/net/201305/1508863.shtm 比特幣的社會實驗：只有相信邏輯才能活在未來]《中國企業家網》| 2013-05-27</ref>
* 2012年12月第一次減半
* 第二次發生減半的實際時間是2016年7月9日 16:46:13 UTC<ref>[https://blockchain.info/block-height/420000 Bitcoin Block Explorer - BTC.com]以2016年6月16日網頁右下方資料顯示</ref><ref>{{cite web|url=http://finance.sina.com.cn/roll/2016-07-10/doc-ifxtwitr1869597.shtml|title=比特币宣布第二次产量减半 业界认为影响不大|work=finance.sina.com.cn}}</ref>，每個資料塊中新發行的比特幣目前為12.5個

== 挖礦难度 ==
為了使得資料塊產生的速度維持在大約每十分鐘一個，產生新資料塊的難度會定期調整。

如果資料塊產生的速度加快了，那麼就提高挖礦難度；如果資料塊產生速度變慢了，那麼就降低難度。比特幣系統在每隔2016個資料塊被產出後（約兩週的時間），會以最近這段時間的資料塊產生速度，自動重新計算接下來的2016個資料塊之挖礦難度。

而難度基本上就決定了一個有效的資料塊標頭（{{Lang-en|Block Header}}）的 [[SHA-2|SHA-256]] 雜湊值應小於一定值，也就是說該雜湊值必須要恰好落在目標區間之內才算有效，當目標區間越小就意味著命中機率越低。

換句話說就是挖礦的難度越高。由於[[特殊應用積體電路|ASIC]]計算設備的爆炸式加入，目前挖礦難度呈現[[幾何級數|幾何級數]]的上升，目前平均每15天增加50%難度{{FACT}}，讓普通個人挖礦者的挖礦工作變得異常困難。

== 挖礦設備 ==
最早，比特币礦工都是通过[[Intel|Intel]]或[[AMD|AMD]]的[[CPU|CPU]]产品来挖礦。但由于挖礦是运算密集型应用，且随着挖礦人数与設備性能的不断提升难度逐渐增加，现在使用CPU挖礦早已毫无收益甚至虧損。

{{As of| 2012}}，从2013年第一季度后，礦工逐渐开始采用[[GPU|GPU]]或[[FPGA|FPGA]]等挖礦設備<ref name="bitcoinmag-butterfly">{{Cite web|url=http://bitcoinmagazine.net/bitpay-breaks-daily-volume-record-with-butterfly-asic-mining-release/|title=Bitpay Breaks Daily Volume Record with Butterfly ASIC mining release|publisher=Bitcoin Magazine|deadurl=yes|archiveurl=https://web.archive.org/web/20170918110522/http://bitcoinmagazine.net/bitpay-breaks-daily-volume-record-with-butterfly-asic-mining-release/|archivedate=2017-09-18}}</ref>。同时，[[特殊應用積體電路|ASIC]]設備也在2013年中旬大量上市<ref name="bitcoinmag-butterfly" />。

从2013年7月起，全网算力由于ASIC設備大量投入运营呈现直线上涨，以2013年7月的平均算力计算，所有CPU挖礦設備均已经无法产生正收益，而FPGA設備也接近无收益。2013年9月平均算力估算，现有的针对个人开发的小型ASIC挖礦設備在未来1-2个月内也接近无正收益。大量算力被 5 THash/s以上的集群式ASIC挖礦設備獨佔。个人挖礦由于没有收益，几乎被挤出挖礦群体。有一些比特幣礦工則集資在某些可取得低價電力的地方興建[[機房|機房]]安裝大批挖礦設備進行挖礦。

部份比特幣礦工為省下自己挖礦的成本，將挖礦程式製作成[[惡意程式|惡意程式]]，在網路上感染其他人的電腦，來替自己挖礦。<ref>[https://technews.tw/2017/12/07/websites-use-your-cpu-to-mine-cryptocurrency-even-when-you-close-your-browser/ 懷疑電腦越來越慢？有 2,500 個網站會偷用你的電腦挖礦，就算關掉瀏覽器也一樣]</ref>

== 礦池 ==
由於比特幣全網的運算水準在不斷的呈指數級別上漲，單個設備或少量的[[算力|算力]]都無法在比特幣網路上獲取到比特幣網路提供的[[區塊獎勵|區塊獎勵]]。在全網算力提升到了一定程度後，過低的獲取獎勵的概率。促使一些「bitcointalk」上的[[極客|極客]]開發出一種可以將少量[[算力|算力]]合併聯合運作的方法，使用這種方式建立的網站便被稱作「礦池」（Mining Pool）。

在此機制中，不論個人礦工所能使用的運算力多寡，只要是透過加入礦池來參與挖礦活動，無論是否有成功挖掘出有效資料塊，皆可經由對礦池的貢獻來獲得少量比特幣獎勵，亦即多人合作挖礦，獲得的比特幣獎勵也由多人依照貢獻度分享。

截止2017年11月，全球算力排名前五的比特币礦池有：AntPool、BTC.com 、BTC.TOP、ViaBTC、F2Pool，目前全球约70%的算力在中国礦工手中<ref>{{Cite web|url=https://btc.com/stats/pool?pool_mode=month3|title=Pool Stats - BTC.com|accessdate=2017-11-08|last=BTC.COM|work=btc.com|language=en}}</ref>。

== 手續費 ==
比特幣礦工會對大部分交易收取少量費用，其主要目的是防止有人大量發送無聊的小額交易，浪費網路資源。当前每筆交易的手續費大部分是฿ 0.0001/KB （即0.0001btc/KB） ，实际上0.9.0版本之後将默认手續費降為฿ 0.00001/KB （即0.00001 btc/KB）。<ref>{{cite web|url=https://bitcoin.org/en/release/v0.9.0|title=Bitcoin Core version 0.9.0 released|work=bitcoin.org}}</ref>因為大部分交易占用的数据量都小于1千字节，所以一般情况下฿ 0.00001（即0.00001 btc/KB）的手續費就足够了。同時，在將來比特幣區塊獎勵較少時，手續費將成為礦工收入的主要來源，比特幣的手續費會與該交易佔用的位元組數相關。 

目前比特幣系統中手續費的計算標準並非強制性的，因此使用者也可以在交易進行時不給予任何手續費，但大多數礦工在組建資料塊時通常會優先考慮帶有較高手續費的交易，以便在挖礦成功時能獲得較高的報酬，因此無附帶任何手續費的交易，可能會需要等待較長的時間才能被處理並納入區塊鏈中。

此外，現在因為區塊容量上限「1MB」以及近期比特幣交易量大增的因素，手續費大幅上漲，而且等待交易被確認的時間也變長了許多。
