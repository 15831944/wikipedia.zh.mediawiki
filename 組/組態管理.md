'''組態管理'''（{{lang-en|Configuration Management}}，簡稱 CM），或譯為'''配置管理'''，是一個建立[[系統工程|系統工程]]的過程，用來建立與維持一個產品，讓這個產品的效能、功能，以及它所要求的物理特性，在它的生命週期中，都能保持穩定與一致性。

在[[資訊|資訊]]與[[電信|電信]]的領域，組態管理、或組態控制（configuration control）擁有以下的意涵:

#從一特定[[系統|系統]]壽命的開發期到營運期，憑藉針對[[自動化資料系統|自動化資料系統]]的[[硬體|硬體]]、[[軟體|軟體]]、[[韌體|韌體]]、[[文件|文件]]、[[測試|測試]]、[[測試設備|測試設備]]、以及[[測試文件|測試文件]]所做的變更的管制，進行安全特點與安全保證的管理，[[原始碼|原始碼]]管理、或[[版本控制|版本控制]]是其中的一部份；
#系統[[生命週期|生命週期]]內從頭到尾，針對硬體、軟體、韌體、以及文件所做的變更（包括其衍生出的紀錄）的管制；
#針對[[複雜系統|複雜系統]]的發展的管制、與適應。是一種規則，能讓逐步成形的軟體產品持續處於控制之下，因此能夠滿足品質，延緩限制。[[软件配置管理|軟體形態管理]]（或簡稱[[SCM|SCM]]）可劃分為二個區域，SCM 的第一個（較為早期的）區域，是關於在軟體開發方案期間所產生的實體的儲存，有時會交付[[構件倉庫管理|構件倉庫管理]]（Component Repository Management，簡稱[[CRM|CRM]]）。SCM 的第二個區域，是關於這些實體的生產、亦或變更的活動，[[工程支援|工程支援]]一詞也經常關聯於這第二個區域；
#當建立了一個（電信、或資訊系統的）[[形態|形態]]之後，針對該形態、或與其相關聯的系統構件所做的評估、與變更核准；
#針對[[分散佇列雙匯流排|分散佇列雙匯流排]]（Distributed Queue Dual Bus，簡稱DQDB）型式的網路，保證一特定DQDB網路的所有節點上的資源，都會被安裝到一個正確的[[雙匯流排拓蹼|雙匯流排拓蹼]]（dual-bus topology）的功能。這些被控管的功能，包括匯流排的表頭資料（Head of Bus，簡稱[[HOB|HOB]]）、外部計時來源、以及內定的[[訊槽產生器|訊槽產生器]]（Slot generator）等功能；

== 軟體組態管理（SCM）的產品 ==

更完整的列表參見[[版本控制軟體列表|版本控制軟體列表]]。

* Cfengine, administers and configures computers according to high-level language configuration files
* [http://www.intland.com  CodeBeamer],a web-based collaborative development tool including tracker with workflow, CMDB, document management, forum, wiki,  source code browsing functions, and Escalation Management（Optional）,  supports Eclipse, ANT, CVS, Subversion, Git, Mercurial, etc. 
* KONFIG CM, [http://www.auto-trol.com/]the total configuration management solution for software, hardware, data and all physical items. 
* LCFG, a large scale UNIX configuration system 
* [http://reductivelabs.com/projects/puppet Puppet], administers and configures computers according to high-level configuration files 
* [http://open.controltier.com/ ControlTier]open source configuration management and process automation tools for deploying multi-tier enterprise applications. 
* [http://www.netdirector.org/ NetDirector]NetDirector is an open source configuration management tool for managing open source applications running on many variants of Linux. Unique features include the ability to simultaneously push changes out to many different platforms. 
* [http://www.isisetup.ch/ IsiSetup] provides a commandline interface to manage your system configuration files using a revision control tool. It runs on most Debian-based distributions and integrates well into the Debian package management system. 
* [http://www.changelogic.com/ Changelogic] enables web based branch, task and release management. It supports CVS and Subversion for underlying version control. 
* [http://www.configurationkit.com/ Configuration Management Kit] a collection of templates, draft plans and similar resources to support CM implementation.. 
* [http://www.microsoft.com/technet/itsolutions/cits/mo/sman/dcm.mspx SMS 2003 Desired Configuration Monitoring] a free download to enable definition of configuration items, automated configuration compliance auditing and compliance reporting 
* [http://www.mercury.com/us/products/business-availability-center/universal-cmdb Mercury Universal CMDB], a toolset consisting of a business/service data model with built-in auto-discovery of configurations dependencies, visualization and business services mapping, and capabilities to track and measure configuration changes.
