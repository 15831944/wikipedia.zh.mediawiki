《'''致命女人'''》（{{lang-en|''Why Women Kill''}}）是一部[[美國|美國]][[黑色幽默|黑色喜劇]][[網絡劇集|網絡劇集]]。由{{le|馬克·切利|Marc Cherry}}主創，將在{{le|CBS All Access}}播出。

== 概要 ==
《致命女人》講述了三個年代的女主角，分別是60年代的家庭主婦、80年代的社交名媛以及2018年的律師，她們都需要面對婚姻內的不忠行為。這部劇集將表現女性的社會角色隨時間變遷，但她們對背叛婚姻的態度沒有改變<ref name="SeriesOrder">{{cite web |last1=Andreeva |first1=Nellie |title=CBS All Access Orders Infidelity Dramedy Series ‘Why Women Kill’ From Marc Cherry & Imagine TV Studios |url=https://deadline.com/2018/09/cbs-all-access-orders-infidelity-dramedy-series-why-women-kill-from-marc-cherry-imagine-tv-studios-1202470030/ |website=[[Deadline_Hollywood|Deadline Hollywood]] |accessdate=2018-12-14 |date=2018-09-24|language=en-US}}</ref>。

=== 開發 ===
2018年9月24日，{{le|CBS All Access}}宣佈給予《致命女人》直接預訂。該劇集由{{le|馬克·切利|Marc Cherry}}主創，他與[[布萊恩·葛瑟|布萊恩·葛瑟]]、弗蘭茜·卡爾弗、邁克爾·哈內和明迪·舒爾希斯一同擔任執行製片人。製作公司包括[[Imagine_Entertainment|幻想娛樂]]和{{le|CBS電視工作室|CBS Television Studios}}<ref name="SeriesOrder"/><ref>{{cite web |last1=Goldberg |first1=Lesley |title=Marc Cherry Dark Comedy 'Why Women Kill' Nabs CBS All Access Order |url=https://www.hollywoodreporter.com/live-feed/marc-cherry-dark-comedy-why-women-kill-nabs-cbs-all-access-order-1146453 |website=[[The_Hollywood_Reporter|The Hollywood Reporter]] |accessdate=2018-12-14 |date=2018-09-24|language=en-US}}</ref><ref>{{cite web |last1=Otterson |first1=Joe |title=CBS All Access Orders Drama Series From ‘Desperate Housewives’ Creator |url=https://variety.com/2018/tv/news/cbs-all-access-marc-cherry-why-women-kill-1202954481/ |website=[[Variety_(magazine)|Variety]] |accessdate=2018-12-14 |date=2018-09-24|language=en-US}}</ref>。12月10日，有報道稱劇集從[[加利福尼亞州|加利福尼亞州]]獲得840萬美元的[[退稅|退稅]]<ref>{{cite web |last1=Patten |first1=Dominic |title=Patrick Stewart’s ‘Star Trek’ Series, ‘Mayans M.C.’ & ‘Why Women Kill’ Score Tax Credits From California |url=https://deadline.com/2018/12/patrick-stewart-star-trek-picard-series-mayans-m-c-orville-why-women-kill-california-film-tax-credits-1202516860/ |website=Deadline Hollywood |accessdate=2018-12-14 |date=2018-12-10|language=en-US}}</ref>。
