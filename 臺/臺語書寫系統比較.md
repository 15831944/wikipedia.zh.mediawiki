==漢羅合用==
{{main|漢羅台文}}

簡稱漢羅，係將臺閩漢字與上述羅馬字方案之一合用，將語音所對應之漢字不明者，直接以羅馬字書寫語音，在1980年代後期開始流行。<REF name="AutoS7-39"> 開放時期ê文字主張：「漢羅合用字」成做主流，梁淑慧[http://taigu.fhl.net/Godspeak/Godspeak17.html]，2006.06.04，台語信望愛</ref>最常見的模式為漢字與[[白話字|白話字]]或者[[臺灣閩南語羅馬字拼音方案|臺羅字]]一同書寫。這種書寫模式與日語的[[漢字假名混寫文|漢字假名合寫文]]、[[韓語|韓語]]廢漢字前的[[漢諺混寫|漢諺合寫]]有異曲同工之妙，只是在文字上並非自創，而是借用西方拼音文字。

== 電腦處理 ==
[[File:Pa-khek-le_Kau-hoe.jpg|thumb]]東門[[巴克禮紀念教會|巴克禮紀念教會]].]]
===輸入法===
{{seealso|鍵盤佈局}}
多數[[键盘布局|键盘布局]]及[[中文輸入法|輸入法]]皆可用於輸入[[拉丁文字|拉丁文字]]或[[漢字|漢字]]。一些可免費使用，一些為商用。

[[閩南語|閩南語]]族的RFC 3066代碼注冊為nan。<REF name="AutoS7-40">RFC 3066 Language code assignments</ref>臺灣話可用「nan-TW」表示。

使用漢字書寫臺灣話時，一些人創立他們認為不會出現的「新」字元以直接使用或假借已有的字；這些取自[[粵語白話文|粵文]]、[[喃字|喃字]]、[[韓文漢字|韓文漢字]]和[[日本漢字|日本漢字]]中的類似用法。這些通常沒被[[Unicode|Unicode]]（或者相應的ISO/IEC 10646：[[通用字符集|通用字符集]]）收錄，給計算機處理帶來了問題。

所有白話字所需的拉丁字元可通過[[Unicode|Unicode]]（或者相應的ISO/IEC 10646：[[通用字符集|通用字符集]]）呈現出來，使用預組或組合用（附加符號）字元。

==={{IPA|[ɔ]}}音的輸入法===
{{main|O͘}}

2004年6月之前，與O相似但開口更大的元音{{IPA|[ɔ]}}使用「向右上方的點」書寫，但沒被分配編碼。通常的變通手法是使用（單獨的）字元「[[間隔號|middle dot]]」（U+00B7，''·''）或者更少見的組合用字元「dot above」（U+0307）。鑒于這些遠離了初衷，自1997年起相關的提議被提交給主管ISO/IEC 10646的ISO/IEC[[工作組|工作組]]——即[https://web.archive.org/web/20070628163947/http://anubis.dkuug.dk/JTC1/SC2/wg2/ ISO/IEC JTC1/SC2/WG2]——以為新的組合用附加符號「dot above right」分配編碼。現在被官方分配給U+0358（參見文-{}-件[http://www.evertype.com/standards/la/taioan.html N1593]、[https://web.archive.org/web/20070311082219/http://anubis.dkuug.dk/JTC1/SC2/WG2/docs/n2507 N2507]、[https://web.archive.org/web/20060928065416/http://anubis.dkuug.dk/JTC1/SC2/WG2/docs/n2628 N2628]、[http://std.dkuug.dk/jtc1/sc2/wg2/docs/n2699 N2699]及[http://std.dkuug.dk/JTC1/sc2/wg2/docs/N2770r.pdf N2770]）。已有字型支援這個字元，如[[Charis_SIL|Charis SIL]]。
