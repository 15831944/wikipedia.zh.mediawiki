在[[專案管理|專案管理]]（project management）領域，'''專案時程表'''（schedule）會列出[[项目|專案]]的[[里程碑|里程碑]]（milestones）、[[活動|活動]]（activities）、與{{Internal link helper/en|交付成果|Deliverable}}（deliverables），並且會有預定開始與結束時間。相關項目通常由專案資源分配、預算、工作時程、依賴性、與已排訂日期的事件來建立。專案排程表通常使用做為[[專案管理|專案管理]]中，{{Internal link helper/en|專案規劃|Project_planning}}、與{{Internal link helper/en|專案組合管理|Project_portfolio_management}}（project portfolio management）的一部分。專案管理時程表中元素通常與[[工作分解结构|工作分解結構]]（WBS）中{{Internal link helper/en|最底層的元件|Work_breakdown_structure#Terminal_element}}、{{Internal link helper/en|工作說明書|Statement_of_work}}或{{Internal link helper/en|合約資料需求清單|Contract_data_requirements_list}}一致。

== 概觀 ==
在很多的產業中（例如工程與建築），通常會視專案規模的大小，聘請專職專案排程者或是排程團隊來開發及管理專案時程表。雖然時程管控的技巧發展相當成熟<ref>[http://www.mosaicprojects.com.au/PDF_Papers/P042_History%20of%20Scheduing.pdf A Brief History of Scheduling (PDF)] {{Wayback|url=http://www.mosaicprojects.com.au/PDF_Papers/P042_History%20of%20Scheduing.pdf |date=20150518071346 }}</ref> ，發展出不同的管理技巧，在不同的產業的應用程度上不太一致。 [[:en:AACE_International|工程成本促進協會]]（AACE）、[[:en:Project_Management_Institute|專案管理協會]]（PMI）<ref>{{cite book|title=A Guide to The Project Management Body of Knowledge|author=Project Management Institute|authorlink=项目管理协会|publisher=Project Management Institute|year=2003|isbn=1-930699-45-X|edition=3rd}}
</ref> 、與[[美國聯邦政府|美國聯邦政府]]（US Government）（針對採購<ref>{{cite web|url=https://acc.dau.mil/CommunityBrowser.aspx?id=37441&lang=en-US|title=Scheduling Guide for Program Managers|accessdate=26 Feb 2014|date=2001|publisher=Defense Systems Management College Press|website=Defense Acquisition University|archive-url=https://web.archive.org/web/20140302222827/https://acc.dau.mil/CommunityBrowser.aspx?id=37441&lang=en-US|archive-date=2014-03-02|dead-url=yes}}</ref> 和會計<ref>{{cite web|url=http://www.gao.gov/products/GAO-16-89G|title=GAO Schedule Assessment Guide, Best Practices for Project Schedules|accessdate=3 Jun 2016|date=December 2015|publisher=General Accounting Office|website=General Accounting Office}}
</ref>） 等單位為推廣時程管控的標準化與最佳實務而努力。

當然，專案管理並非只能運用在限定的專案領域裡，一般人也可以運用專案管理技巧在自己生活周遭的組織上。例如下面案例：
* 房屋裝修專案
* 持續追蹤所有家庭成員的活動
* 輔導團隊
* 渡假計畫
* 結婚計畫
部分[[專案管理軟體|專案管理軟體]]會提供專案範本、清單、與時程表範本，來方便使用者建立各自需求的專案時程。

== 方法 ==
在專案時程建立初期，專案經理需要透過工作分解結構（WBS）逐一分析每項任務所耗用的資源，並且一個可以運用資源可用性（時間、數量）的清單。如果當中有某項元件缺少，可以用團隊決策的估計方法建立，如{{Internal link helper/en|寬帶德菲法|Wideband_delphi}}（團體出牌法）。這樣做的原因是因為時程原本就是一種預估：時程表中的每一個日期或時間都是預估值，如果該預估的時間沒有預排的人員執行任務，那該時程表預估的時間就變得不準確。<ref>Stallman, Greene, [http://www.stellman-greene.com/aspm/content/view/18/38/ Applied Software Project Management] {{webarchive|url=https://web.archive.org/web/20130624124503/http://www.stellman-greene.com/aspm/content/view/18/38/ |date=2013-06-24 }}  O'Reilly press, Nov 2005</ref>

建立一個專案時程表，需要透過以下步驟完成：<ref>{{Cite web|url=https://www.geniusproject.com/guide/beginner-guide-project-scheduling/getting-started|title=Beginner's Guide to Project Scheduling|accessdate=15 May 2017|publisher=Genius Project}}</ref>
* 專案範疇（範圍）
* 活動排序 
* 五個週期的工作群組（概念、定義及計畫、發行、效能、結束）
* 工作相依圖 
* 關鍵路徑分析 
* 專案里程碑
為了建立一套健全的專案時程表，必須要切中以下重點：<ref>Cutting, Thomas, [http://www.pmhut.com/cultivating-a-healthy-project-schedule Cultivating a Healthy Project Schedule], PM Hut (Last accessed 8 November 2009).</ref>
* 該時程表必須要定期更新（每周更新一次較佳）。
* [[挣值管理|EAC（預估完成工時]]）數值必須要等於預估工時基準值。
* 剩餘的工作必須要在專案團隊成員之間被適當分配（也應把休假考慮進去）
需要密切關注的是專案時程表的結構要引用工作分解結構（WBS）或可交付成果的索引，使用定義在WBS中的分解架構或範本來描述與產出可交付成果所需進行的活動項目。<ref>{{Cite web|url=http://stsc.hill.af.mil/resources/tech_docs/gsam4.html|title=Guidelines for Successful Acquisition and Management of Software-Intensive Systems, 7.2.1 Activity Definition|accessdate=15 Jan 2016|publisher=USAF Software Technology Support Center|deadurl=yes|archiveurl=https://web.archive.org/web/20121230143752/http://www.stsc.hill.af.mil/resources/tech_docs/gsam4.html|archivedate=2012-12-30}}</ref>

對時程表的評估可以建立在時程表開發品質和時程管理品質兩個面向。<ref>{{Cite web|url=http://www.dcma.mil/policy/200-1/PAM-200-1.pdf|title=DCMA-EA PAM 200.1 Program Analysis Pamphlet|accessdate=24 May 2016|date=Oct 2012|deadurl=yes|archiveurl=https://web.archive.org/web/20151101044222/https://www.dcma.mil/policy/200-1/PAM-200-1.pdf|archivedate=2015年11月1日}}</ref>
