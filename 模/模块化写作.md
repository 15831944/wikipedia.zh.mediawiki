模块化写作（英文：'''Topic-based authoring''' 或 '''Topic-based writing'''）是一种以'''主题'''（英文：'''Topic'''）为模块进行内容创作的写作方式，是编写各类技术文档是最常用的方式之一。每个主题是一个独立的内容模块，包含标题和内容（有时可以只有标题）向读者提供一段完整的信息。各个主题之间可以互相嵌套，也可以独立存在。根据[[结构化资讯标准促进组织|结构化资讯标准促进组织]]（OASIS）发布的[[达尔文信息类型架构|达尔文信息类型架构]]（DITA）规范，主题可以分为概念、任务、参考、术语表和故障排查五种类型<ref name="Darwin Information Typing Architecture (DITA) Version 1.3 Part 3: All-Inclusive Edition">{{cite web |author=DITA Technical Committee |title=Technical content: Document types |url=http://docs.oasis-open.org/dita/dita/v1.3/os/part3-all-inclusive/archSpec/technicalContent/dita-technicalContent-InformationTypes.html#dita_technicalContent_InformationTypes |publisher=OASIS Standard}}</ref>。模块化写作通常使用基于 [[XML|XML]] 语言的编辑工具来创作内容模块。作者可以使用[[内容管理系统|内容管理系统]]来存储这些内容模块，并根据需求对内容模块进行嵌套和组合，以此来生成不同的文档或其他形式的信息产品。<ref>{{cite web |title=基于主题的模块化写作——DITA特点浅析（三） |url=http://www.tc-china.org/article/tools/3073/ |website=TC 互联}}</ref>。
[[达尔文信息类型架构|达尔文信息类型架构]]是一种旨在帮助作者创建模块化内容的标准，该标准由[[结构化资讯标准促进组织|结构化资讯标准促进组织]]的 DITA 技术委员会创建并维护<ref>{{cite web |title=OASIS Darwin Information Typing Architecture (DITA) TC |url=https://www.oasis-open.org/committees/tc_home.php?wg_abbrev=dita}}</ref>。
