「設計概要」（{{Lang-en|Design brief}}）是指由個人或團隊（設計師或是設計團隊）為需求對象所撰寫的專案開發文件，它將專案中包含的任何產品或工作內容（功能性與美感）、時程以及預算等具體地描繪出交付內容與規模。設計概要可以運用在許多不同範疇的專案上，像是建築、室內設計以及工業設計等。設計概要同時也可以用於評估成品，以及確保專案能夠被有效追蹤並在預算內完成。每間公司對於設計概要的使用程度不盡相同，不過業界對於設計過程中的責任歸屬的日益重視，因此很多人認為它能派上用場。設計概要的內容經常隨著專案的演進而調整，通常它是在專案初期階段由客戶以及設計師所簽核。

一份設計綱要文件可能會使用下列格式：

# 封面
# 目錄
# 沿革
#* 公司沿革
# 公司簡介
#* 規格
#* 設計師簡介
#* 公司名稱
#* 過往成就
# {{link-en|問題陳述|Problem Statement}}
#* 問題描述
#* 限制
#* [[預算|預算]]
#* 時程
#* Needs of the Problem
# [[目標|目標]]
#* 預期達成
#** 期限
# 方案分析
#* [[風險|風險]]/[[Economics|優勢]]
#* 計畫解決方案
#* {{link-en|草圖|Sketch (drawing)}}
# 大綱
#* 評估
#* 總結/摘要
