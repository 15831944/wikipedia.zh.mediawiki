'''对[[GitHub|GitHub]]的审查和封锁'''经常在多个国家发生，例如[[中华人民共和国|中国]]、[[俄罗斯|俄罗斯]]、[[印度|印度]]、[[土耳其|土耳其]]等。审查和封锁方式包括但不限于[[DNS污染|DNS污染]]、[[中间人攻击|中间人攻击]]等。但是，由於封鎖引發了用戶和科技公司的強烈反應，這些國家最終都解除了對GitHub的封鎖。

==背景==
[[GitHub|GitHub]]是一个基于[[Git|Git]]的源代码托管网站，主要用于存储[[源代码|源代码]]并用于项目管理。截止到2015年，GitHub已有超过1000万用户和2.4亿个Git仓库。<ref>{{cite web |url=https://github.com/about/press |title=GitHub Press Info |website=github.com |publisher=Github |accessdate=2015-06-27 |archive-url=https://web.archive.org/web/20160517134612/https://github.com/about/press |archive-date=2016-05-17 |dead-url=no }}</ref>此外，GitHub还提供了免费的{{link-en|粘贴箱|pastebin}}服务（[[GIST|Gist]]）和免费的网络空间GitHub Pages。GitHub许可协议禁止滥用GitHub的各项服务，保留了删除内容的权力。<ref>{{cite web|title=GitHub Terms of Service|url=https://help.github.com/articles/github-terms-of-service/|website=GitHub|accessdate=2015-06-27|archive-url=https://web.archive.org/web/20150624142711/https://help.github.com/articles/github-terms-of-service/|archive-date=2015-06-24|dead-url=no}} (Specifically terms A8 and G7)</ref>此外，如果GitHub收到[[數字千年版權法|DMCA]]删除通知，GitHub也会删除项目。<ref>{{cite web|title=DMCA Takedown Policy|url=https://help.github.com/articles/dmca-takedown-policy/#b-what-about-forks-or-whats-a-fork|website=GitHub|accessdate=2015-06-27|archive-url=https://web.archive.org/web/20150701020203/https://help.github.com/articles/dmca-takedown-policy/#b-what-about-forks-or-whats-a-fork#b-what-about-forks-or-whats-a-fork|archive-date=2015-07-01|dead-url=no}}</ref>

===DNS污染===
2013年1月20日，[[中国大陆|中國大陸]]政府的[[防火長城|防火長城]]利用[[域名污染|域名污染]]和關鍵詞過濾等手段封鎖GitHub，令中國大陸的用戶無法直接進入。針對政府對GitHub的封鎖行動，[[李開復|李开复]]在[[新浪微博|新浪微博]]抗議，迅速引起網民的關注，該訊息更在三小時内被轉發逾3萬2千次<ref>{{cite news|url=http://nandu.oeeee.com/nis/201301/23/6267.html|title=程序员叫苦 李开复质疑|date=2013-01-23|publisher=南方都市报|accessdate=2013-01-23|deadurl=yes|archiveurl=https://web.archive.org/web/20130126133757/http://nandu.oeeee.com/nis/201301/23/6267.html|archivedate=2013年1月26日}}</ref><ref>{{cite news |url=http://www.scmp.com/news/china/article/1134717/programmers-angry-over-blocking-github-code-sharing-site |title=Programmers angry over blocking of GitHub code-sharing site |date=2013-01-24 |publisher=南华早报 |accessdate=2013-01-24 |archive-url=https://web.archive.org/web/20130126212832/http://www.scmp.com/news/china/article/1134717/programmers-angry-over-blocking-github-code-sharing-site |archive-date=2013-01-26 |dead-url=no }}</ref>。2013年1月23日，GitHub被解封，事件平息。

後來，Gist又被防火長城封鎖，但GitHub其他服務未受影響。<ref>{{Cite web |url=http://www.solidot.org/story?sid=41709 |title=存档副本 |access-date=2016-07-02 |archive-url=https://web.archive.org/web/20150924103633/http://www.solidot.org/story?sid=41709 |archive-date=2015-09-24 |dead-url=no }}</ref>

===中间人攻击===
2013年1月26日，有中国大陆的用户在访问GitHub时发现证书无效，经检查发现，GitHub的证书变为了一自签署的X.509证书，生成时间为2013年1月25日14时29分12秒，有效期一年，故有人推测GitHub疑似遭到了[[中间人攻击|中间人攻击]]。攻击持续了约一个小时后停止，访问恢复正常。<ref>{{cite news |url=http://www.solidot.org/story?sid=33284 |title=中国国家防火墙对GitHub进行了中间人攻击 |date=2013-01-26 |publisher=solidot |accessdate=2013-01-26 |deadurl=yes |archiveurl=https://web.archive.org/web/20130128122326/http://www.solidot.org/story?sid=33284 |archivedate=2013-01-28 }}</ref>

===DDoS攻击===
==== 抢票插件风波 ====
2013年1月15日晚，GitHub遭到疑似DDoS攻击，管理员查询日志后得知，攻击来自中国的一个[[12306|12306]]抢票插件。因为该插件代码本身的缺陷，加上插件作者低估了[[春运|春运]]抢票的人数，构成了DDoS攻击。<ref>{{cite web|url=http://tech.sina.com.cn/i/csj/2013-01-16/19367984516.shtml|title=12306抢票插件拖垮美国代码托管站Github|date=2013-01-16|access-date=2016-07-02|archive-url=https://web.archive.org/web/20160510183731/http://tech.sina.com.cn/i/csj/2013-01-16/19367984516.shtml|archive-date=2016-05-10|dead-url=no}}</ref>1月20日，GitHub在中国遭到屏蔽。有评论认为，该抢票插件是导致GitHub在中国被屏蔽的原因。<ref>{{cite news|url=http://www.pcworld.idg.com.au/article/451381/github_blocked_china_ticket_snatching_plugins_seen_possible_cause/|title=GitHub blocked in China, 'ticket snatching' plugins seen as possible cause|date=2013-01-22|publisher=PC World|access-date=2016-07-02|archive-url=https://web.archive.org/web/20170110211046/http://www.pcworld.idg.com.au/article/451381/github_blocked_china_ticket_snatching_plugins_seen_possible_cause/|archive-date=2017-01-10|dead-url=no}}</ref>

==== 旁觀者攻擊 ====
{{See also|旁观者攻击|大炮 (中國網路審查)}}
GitHub从2015年3月26日凌晨2時左右（[[协调世界时|世界協調時間]]，以下簡稱為UTC）起遭到了持续的[[阻斷服務攻擊|DDoS]]攻击。<ref>{{Cite web |url=https://status.github.com/messages |title=GitHub System Status Messages |access-date=2016-07-02 |archive-url=https://web.archive.org/web/20160702045342/https://status.github.com/messages |archive-date=2016-07-02 |dead-url=no }}</ref>GitHub在其[[部落格|部落格]]称，这次攻击是GitHub历史上最严重的一次DDoS攻击，<ref name="GH blog">{{Cite web |url=https://github.com/blog/1981-large-scale-ddos-attack-on-github-com |title=Large Scale DDoS Attack on github.com |accessdate=2015-03-31 |date=2015-03-27 |publisher=GitHub's blog |language=en |archive-url=https://web.archive.org/web/20150328034203/https://github.com/blog/1981-large-scale-ddos-attack-on-github-com |archive-date=2015-03-28 |dead-url=no }}</ref>

GitHub堅信攻擊者的目的在於迫使GitHub刪除特定類型的內容。<ref name="GH blog" /><ref name="iThome.com.tw" /><ref>{{Cite web |url=http://www.solidot.org/story?sid=43500 |title=GitHub DDoS 攻击还在继续，百度否认与其有关 |accessdate=2015-03-30 |date=2015-03-28 |publisher=奇客[[Solidot|Solidot]].org |language=zh-cn |deadurl=yes |archiveurl=https://web.archive.org/web/20150330001407/http://www.solidot.org/story?sid=43500 |archivedate=2015-03-30 }}</ref>第三方研究者指出，此次攻击采用了HTTP劫持，[[百度|百度]]的[[JavaScript|JS脚本文件]]被[[中间人攻击|中间人]]植入了攻击GitHub的代码，其功能是每隔2秒加载一次[[Greatfire.org|GreatFire]]或[[纽约时报|纽约时报中文网]]的帳號主頁。<ref>{{cite web|url=http://drops.wooyun.org/papers/5398|publisher=乌云知识库|title=百度统计js被劫持用来DDOS Github|author=insight-labs|access-date=2016-07-02|archive-url=https://web.archive.org/web/20160922173820/http://drops.wooyun.org/papers/5398|archive-date=2016-09-22|dead-url=no}}</ref>百度已否认自身产品存在安全问题。<ref>{{cite web|url=http://weibo.com/3326069452/Caw7wsZ1L|title=百度安全攻防实验室的微博|author=百度在线網络技术（北京）有限公司}}</ref>。这次攻击导致GitHub在全球范围内的访问速度下降。<ref>{{cite web|url=https://status.github.com/messages/2015-03-28|title=GitHub System Status|author=GitHub|access-date=2016-07-02|archive-url=https://web.archive.org/web/20170219163334/https://status.github.com/messages/2015-03-28|archive-date=2017-02-19|dead-url=no}}</ref>。

3月28日（[[UTC+08:00|UTC+8]]）起，GitHub在中国大陆十分不稳定，多数情况下无法访问。<ref>{{Cite web |url=http://www.solidot.org/story?sid=43495 |title=Github证实遭到DDoS攻击，HTTP劫持已停止 |date=2015-03-27 |publisher=奇客Solidot |language=zh-cn |access-date=2016-07-02 |archive-url=https://web.archive.org/web/20160323065419/http://www.solidot.org/story?sid=43495 |archive-date=2016-03-23 |dead-url=no }}</ref>截止29日，攻击者共使用了四種DDoS攻擊技術：
#第一輪，利用中國大陸以外的網民與[[突破網路審查|翻牆]]的網民瀏覽被劫持的百度JavaScript檔案，該檔案每2秒向GitHub上的两个頁面發出請求，被GitHub的彈窗警告攔住；
#第二轮，跨網域<img>攻擊，被GitHub检查[[HTTP參照位址|Referer]]攔住；
#第三輪，DDoS攻擊[[GitHub_Pages|GitHub Pages]]；
#第四輪，[[SYN_flood|SYN flood]]，利用[[TCP协议|TCP協定]]缺陷发送大批伪造的TCP連線請求，耗盡GitHub的資源。<ref>{{Cite web |url=http://www.solidot.org/story?sid=43506 |title=对GitHub的大规模DDoS攻击已超过80个小时 |accessdate=2015-03-30 |date=2015-03-30 |publisher=奇客Solidot |language=zh-cn |archive-url=https://web.archive.org/web/20150331103957/http://www.solidot.org/story?sid=43506 |archive-date=2015-03-31 |dead-url=no }}</ref>

[[File:Traceroute-GFW.png|thumb]]追踪[[存活時間|TTL]]来证明中国政府对GitHub发动攻击]]

=== 运营商封锁 ===
[[File:Github_russia_block.png|thumb]]

2014年12月2日，{{link-en|俄联邦电信、信息技术和大众传媒监督局|Roscomnadzor}}因为网站内包含了几种涉及自杀教学的内容而封禁了GitHub。因为GitHub全站使用[[超文本传输安全协议|HTTPS]]连接，无法针对部分页面进行封锁，因此[[互联网服务供应商|互联网服务供应商]]被强制要求封锁整个网站。 实行封锁的运营商包括：Beeline、MTS、MGTS和Megafon。监督局负责人Maxim Ksenzov表示，GitHub被封禁是因为之前的2014年10月10日GitHub未遵守俄罗斯的删除请求。<ref>{{cite web|last1=Lunden|first1=Ingrid|title=Russia Blacklists, Blocks GitHub Over Pages That Refer To Suicide|url=http://techcrunch.com/2014/12/03/github-russia|website=TechCrunch|accessdate=2015-04-01|archive-url=https://web.archive.org/web/20150327101211/http://techcrunch.com/2014/12/03/github-russia/|archive-date=2015-03-27|dead-url=no}}</ref>在2014年10月2日GitHub曾被短暂封锁，直到原始的自杀教程被删除。<ref>{{cite web|last1=Лихачёв|first1=Никита|title=AliExpress, 2ch и GitHub попали в реестр запрещённых сайтов|url=http://tjournal.ru/paper/rkn-github-2ch|website=TJournal|accessdate=2015-04-09|language=ru|trans-title=AliExpress, 2ch and GitHub put on the register of banned sites|archive-url=https://web.archive.org/web/20150425013521/http://tjournal.ru/paper/rkn-github-2ch|archive-date=2015-04-25|dead-url=no}}</ref>

=== 被封禁的内容 ===
2014年3月23日，有人在GitHub上面发布了介绍自杀的手册，内含在俄罗斯内的31种自杀方法，该项目被数个用户[[复刻_(软件工程)|複刻]]。<ref name="amdf-create">{{cite web|title=Create suicide.txt|url=https://github.com/amdf/objidlib/commit/e9c31e97c0ca7863516fa6e21c9f6441453d9149|website=GitHub - amdf/objidlib|accessdate=2015-04-08|archiveurl=https://web.archive.org/web/20150627095452/https://github.com/amdf/objidlib/commit/e9c31e97c0ca7863516fa6e21c9f6441453d9149|archivedate=2015-06-27|deadurl=no}}</ref>在GitHub被封锁之后，很多GitHub用户举报了该项目。该项目于2014年10月2日被建立者删除。{{efn|然而，因为该项目被複刻，并且[[Git|Git]]会记录变更历史，因此内容仍然可以在GitHub内被找到。}}<ref name="amdf-create"/><ref>{{cite web|title=Delete suicide.txt|url=https://github.com/amdf/objidlib/commit/643077865e390f1eccacf9620199772e1e468a4a|website=GitHub - amdf/objidlib|accessdate=2015-04-09|archiveurl=https://web.archive.org/web/20150627095712/https://github.com/amdf/objidlib/commit/643077865e390f1eccacf9620199772e1e468a4a|archivedate=2015-06-27|deadurl=no}}</ref>

=== 回应 ===
按照俄罗斯政府的相关要求，GitHub屏蔽了相关仓库在俄罗斯内的访问，以解除对GitHub的封锁。<!-- Citing its {{tsl|en|terms of service||terms of service}}, GitHub elaborated that "you must not, in the use of the Service, violate any laws in your jurisdiction (including but not limited to copyright or trademark laws)."<ref name=Lunden-TC/>-->GitHub为监督局专门建立了一个名为“roskomnadzor”的仓库，并记录来自监督局的删除通知。在仓库的README文件中，GitHub指出，他们对[[互联网审查|互联网审查]]表示关注，公开透明是为了防止潜在的[[史翠珊效应|史翠珊效应]]。GitHub还指出，发布通知仅仅是为了记录，不会针对仓库内容的观点进行判断。<ref>{{cite web|last1=Geraci|first1=Jesse|title=github/roskomnadzor - README.md|url=https://github.com/github/roskomnadzor/blob/master/README.md#what-is-this|website=GitHub|accessdate=2015-04-01|archive-url=https://web.archive.org/web/20170615120448/https://github.com/github/roskomnadzor/blob/master/README.md#what-is-this#what-is-this|archive-date=2017-06-15|dead-url=no}}</ref>

==印度==
印度根据联邦和州法律来审查网站。互联网服务供应商需要按照{{link-en|信息技术法|Information Technology Act, 2000}}的要求来进行审查。但有评论指出，这些法律规定比较模糊，供应商容易过分审查。<ref>{{cite web|last1=Patry|first1=Melody|title=India: Digital freedom under threat? === 运营商封锁 ===
在2014年12月17日，{{link-en|印度电信部门|Department of Telecommunications}}命令运营商封禁32个网站。<ref name=Saxena-TOI>{{cite web|last1=Saxena|first1=Anupam|title=Pastebin, Dailymotion, Github blocked after DoT order: Report|url=http://timesofindia.indiatimes.com/tech/tech-news/Pastebin-Dailymotion-Github-blocked-after-DoT-order-Report/articleshow/45701713.cms|website=The Times of India|accessdate=2015-04-01|archive-url=https://web.archive.org/web/20150302071856/http://timesofindia.indiatimes.com/tech/tech-news/Pastebin-Dailymotion-Github-blocked-after-DoT-order-Report/articleshow/45701713.cms|archive-date=2015-03-02|dead-url=no}}</ref>命令于2014年12月31日被公开，其中包括GitHub、Gist、[[Vimeo|Vimeo]]、[[互联网档案馆|互联网档案馆]]和多个{{link-en|粘贴箱|pastebin}}服务。<ref name="blue-ZDN">{{cite web|last1=Blue|first1=Violet|title=India blocks 32 websites, including GitHub, Internet Archive, Pastebin, Vimeo|url=http://www.zdnet.com/article/india-blocks-32-websites-including-github-internet-archive-pastebin-vimeo|website=ZDNet|accessdate=2015-04-01|archive-url=https://web.archive.org/web/20150402065723/http://www.zdnet.com/article/india-blocks-32-websites-including-github-internet-archive-pastebin-vimeo/|archive-date=2015-04-02|dead-url=no}}</ref>

封锁令被印度执政党[[印度人民党|印度人民党]]的Arvind Gupta所证实，他还指出封锁令与印度的反恐小组针对[[伊斯兰国|伊斯兰国]]的建议有关。Gupta还指出，配合政府调查的网站会被解除封锁。<ref>{{cite web|last1=Ghoshal|first1=Abhimanyu|title=GitHub, Vimeo and 30 more sites blocked in India over content from ISIS|url=http://thenextweb.com/in/2014/12/31/vimeo-github-30-sites-blocked-india-content-isis|website=The Next Web|accessdate=2015-04-01|archive-url=https://web.archive.org/web/20150404055942/http://thenextweb.com/in/2014/12/31/vimeo-github-30-sites-blocked-india-content-isis/|archive-date=2015-04-04|dead-url=no}}</ref>

2015年1月2日，印度通信部门宣布将解封四个网站，其中有GitHub Gist，并指出一旦剩下的网站遵守印度法律，也会被解除封锁。解释封禁原因时，部长提到“在这些网站中，大多数网站在发布内容之前都不需要任何审核……这些网站经常被用于传递和交流[[吉哈德|吉哈德]]内容……”<ref name=Sharma-TOI>{{cite web|last1=Sharma|first1=Ravi|title=Indian government unblocks Vimeo, Dailymotion, 2 other websites|url=http://timesofindia.indiatimes.com/tech/tech-news/Indian-government-unblocks-Vimeo-Dailymotion-2-other-websites/articleshow/45731251.cms|website=The Times of India|accessdate=2015-04-01|archive-url=https://web.archive.org/web/20150208102100/http://timesofindia.indiatimes.com/tech/tech-news/Indian-government-unblocks-Vimeo-Dailymotion-2-other-websites/articleshow/45731251.cms|archive-date=2015-02-08|dead-url=no}}</ref><!-- Gulshan Rai of the {{tsl|en|CERT-In||CERT-In}} agency of the ministry said that the order came from the Mumbai Additional Chief Metropolitan Magistrate following an interrogation of Arif Majeed, an ISIS recruit.<ref name=Arora-TOI />-->

2015年1月4日，一个GitHub发言人指出，一些用户在访问GitHub时仍然会遇到困难，GitHub已经和印度政府进行接触，但仍然无法确定封禁的具体原因。发言人还指出，使印度的开发者能够重新访问GitHub是他们的高优先级工作，他们“愿意与印度政府合作，建立一套识别和处理非法内容的过滤系统，以恢复网站的正常访问，并确保将来不会再受到影响。”<ref>{{cite web|last1=Orsini|first1=Lauren|title=India Unblocks GitHub, Three Other Websites|url=http://readwrite.com/2015/01/02/india-lifts-internet-block-github|website=readwrite|accessdate=2015-04-01|archive-url=https://web.archive.org/web/20150320205330/http://readwrite.com/2015/01/02/india-lifts-internet-block-github|archive-date=2015-03-20|dead-url=no}}</ref>

=== 影响 ===
据[[印度时报|印度时报]]报道，运营商Vodafone、BSNL、Hathway封锁了GitHub而通过Airtel仍能访问。<ref name=Saxena-TOI/>因为封锁令只告诉运营商封锁哪些网站但未指出如何封锁，因此封锁程度是不同的。封锁是不可靠的，并且似乎发生在多个层面上。即使是同一运营商，封锁方法也有多种，包括[[IP封锁|IP封锁]]、使用[[代理服务器|代理服务器]]、[[DNS污染|DNS污染]]。根据封锁程度不同，突破封锁的方法也不同，例如更换[[域名系统|DNS服务器]]或使用[[代理软件|代理软件]]。<ref>{{cite web|last1=Srikanth|first1=Kaustubh|title=Technical Observations About Recent Internet Censorship In India|url=http://www.huffingtonpost.in/kaustubh-srikanth/technical-observations-ab_b_6421306.html|website=The Huffington Post|accessdate=2015-04-01|archive-url=https://web.archive.org/web/20150521194154/http://www.huffingtonpost.in/kaustubh-srikanth/technical-observations-ab_b_6421306.html|archive-date=2015-05-21|dead-url=no}}</ref>

== 土耳其 ==
[[File:October_9,_2016_GitHub_Turkey_block.jpg|thumb]]
2016年10月8日，在土耳其黑客组织{{le|RedHack|RedHack}}声称入侵土耳其能源部长、[[埃尔多安|埃尔多安]]女婿{{le|阿尔巴伊拉克|Berat Albayrak}}之后，{{le|土耳其信息与通信技术署|Information and Communication Technologies Authority}}命令互联网服务供应商封锁几个[[檔案分享|檔案分享]]网站，包括{{tsl|en|Dropbox||Dropbox}}、[[OneDrive|Microsoft OneDrive]]、[[Google雲端硬碟|Google Drive]]<ref name=ref01>{{cite web|title=Dropbox, Google Drive and Microsoft OneDrive cloud services blocked in Turkey following leaks|url=https://turkeyblocks.org/2016/10/08/google-drive-dropbox-blocked-in-turkey/|website=Turkey Blocks|accessdate=2016-10-09|archive-url=https://web.archive.org/web/20161209235232/https://turkeyblocks.org/2016/10/08/google-drive-dropbox-blocked-in-turkey/|archive-date=2016-12-09|dead-url=no}}</ref>。觀測網路審查情況的{{le|Turkey Blocks|Turkey Blocks}}注意到GitHub於次日上午被封鎖，並且相關的封鎖令被信息与通信技术署逐步發佈出來<ref name=ref01/>。封鎖導致依賴GitHub的軟體（例如[[Font_Awesome|Font Awesome]]、{{tsl|en|Homebrew||Homebrew}}等）無法正常運行。Startup Istanbul活動的參與者也表達了對基礎設施可用性的不滿。在土耳其，#GitHub[[主題標籤|標籤]]成為了Twitter的熱門標籤之一。按照{{le|The Daily Dot|The Daily Dot}}的說法，RedHack組織蓄意通過多種服務來傳播郵件內容，期待土耳其政府將其封鎖，以造成[[史翠珊效應|史翠珊效應]]。18小時後GitHub被解封<ref>{{cite web|last1=Sozeri|first1=Efe Kerem|title=How hacktivist group RedHack gamed Turkey’s censorship regime|url=http://www.dailydot.com/layer8/redhack-gamed-turkey-censorship/|website=Daily Dot|accessdate=2016-10-12|archive-url=https://web.archive.org/web/20161013074044/http://www.dailydot.com/layer8/redhack-gamed-turkey-censorship/|archive-date=2016-10-13|dead-url=no}}</ref>。
