== 遭“后门”黑幕 ==
2010年2月2日，[[瑞星|瑞星公司]]發布《瑞星揭露黑幕：奇虎360给用户装“后门”》的文章，發現360產品在安裝進用戶電腦時，會偷偷開設“[[後門|後門]]”，造成資料泄露隐患。360随后表示已修复该漏洞。但瑞星在其官方页面更新称：据瑞星安全专家分析，在360之后版本（6.1.5.1009）中此“后门”仍然存在。

此爭議之訴訟於2011年11月，由360勝訴<ref>{{cite web|url=http://www.rising.com.cn/about/news/rising/2010-02-02/6617.html | title=瑞星揭露黑幕：奇虎360给用户装“后门” |publisher=瑞星 |date=2010-02-02 |accessdate=2010-02-02| language = zh-hans}}</ref><ref>{{cite web|title=360公开致谢NT Internals已第一时间抢修漏洞|url=http://bbs.360.cn/3229787/34800737.html|publisher=360安全中心|accessdate=2012-12-30|language=zh-hans|deadurl=yes|archiveurl=https://web.archive.org/web/20130131031405/http://bbs.360.cn/3229787/34800737.html|archivedate=2013-01-31}}</ref>。

== 虛假補丁事件 ==
2012年8月1日，有用户反映360安全卫士“漏洞修复”功能中编号为KB360018的“高危漏洞补丁”，并非由[[微软|微软]]公司发布。此补丁会强制在用户电脑内安装[[360安全浏览器|360安全浏览器]]。<ref>{{cite web|url=http://news.mydrivers.com/1/236/236409.htm | title=360假冒发布系统补丁　微软官方或将介入调查 |publisher=驱动之家 |date=2012-08-01 |accessdate=2012-08-01| language = zh-hans}}</ref><ref>http://weibo.com/1709486153/yvbq1e1Ks</ref>
360官方也做出了正式回应，承认KB360018确实并非微软官方补丁，而是由360自行发布的“伪补丁”，初衷是“提供的[[Internet_Explorer|IE]]6内核升级方案”。<ref>{{cite web|url=http://news.mydrivers.com/1/236/236587.htm | title=360紧急下架假冒补丁|publisher=驱动之家 |date=2012-08-03 |accessdate=2012-08-03| language = zh-hans}}</ref>

== 被评测机构AV-C取消测试奖项 ==
2015年4月30日，国际杀毒软件评测机构AV-Comtives（简称AV-C）对中国杀毒软件厂商发起质疑，[[腾讯|腾讯]]和[[百度|百度]]被指使用带有特殊标记的版本参评，成绩最好的360更是被取消了2015年奖项。AV-Comtives、AV-TEST和Virus Bulletin同時取消了360產品於2015年以來獲得的所有認證和評級。在這三間評測機構的聯合公告中，認為奇虎360在測試中使用了[[Bitdefender|Bitdefender]]引擎（中文名比特梵德，简称BD引擎），而奇虎360自主開發的QVM引擎從未被開啟過。但是在奇虎360的市場版本中，都關閉了BD引擎，只使用QVM引擎，降低了安全層級；因此認為奇虎360涉及為了評比而造假。<ref>{{cite news|url=http://www.itnews.com.au/news/chinese-security-vendor-caught-cheating-in-av-test-403418|title=Chinese security vendor caught cheating in AV test|date=2015-05-01|publisher=iTnews}}</ref><ref>{{cite web|last1=Ilascu|first1=Ionut|title=Security Vendor Tencent Optimizes Antivirus for Better Independent Test Results|url=http://news.softpedia.com/news/Security-Vendor-Tencent-Optimizes-Antivirus-for-Better-Independent-Test-Results-480299.shtml|website=Softpedia|accessdate=5 June 2015}}</ref>

360副总裁曲晓东表示，此次事件根本问题是传统杀毒与云安全的评测标准之争，360的参评版本不存在任何造假作弊问题；但由于此评测标准太过落后，360将退出AV-C评测。<ref>{{cite news|title=被评测机构AV-C取消奖项后，360退出不玩了，称标准落后|url=http://news.163.com/15/0503/10/AOMFDV5O00014U9R.html| publisher = [[网易|网易]] |date=2015-05-03|accessdate=2015-05-03| language = zh-cn}}</ref>

== 恶意推广 ==
[[File:360sd-malware-screenshot.png|thumb]]
360以安全卫士、360杀毒、360安全浏览器等软件会通过优化、清理、修复等弹窗或按钮，在未明确提示用户或未经用户许可的情况下，诱导或强制推广安装奇虎360系列软件（“360全家桶”），并将[[浏览器|浏览器]]等默认软件篡改为360系列软件<ref>{{cite web|title=大量用户因捆绑卸载360软件 你的电脑还安全吗？|url=http://gb.cri.cn/44571/2014/10/31/7872s4749272.htm|website=国际在线|accessdate=2017-08-31}}</ref><ref>{{cite web|title=360欺骗用户捆绑安装引众怒|url=http://www.donews.com/net/201301/1718802.shtm|website=Donews|accessdate=2017-08-31}}</ref>。

[[安卓|安卓]]手机用户发现，在打开USB调试的情况下，只要将手机与电脑连接，安装了360安全卫士的电脑就会自动为用户手机装上“360手机安全套装”（通常为360手机卫士、360手机助手、360手机浏览器），安装全过程无任何提示，也无须经过用户许可确认，直至所有软件安装完毕后，才会以弹窗的形式告知用户。大量手机用户还遇到过强制安装360手机助手后自动捆绑安装其他恶意软件，无法卸载、无法使用其它同类软件的问题。对此，北京工商行政管理部门曾针对360安全卫士软件诱导用户使用360浏览器，并通过不兼容、难卸载等方式阻止用户安装和使用其他安全软件的[[不正当竞争|不正当竞争]]行为予以行政告诫<ref>{{cite web|title=捆绑比预装更可怕 360捆绑安装手机助手遭网友投诉|url=http://tech.hexun.com/2014-03-17/163104090.html|website=和讯网|accessdate=2017-08-31}}</ref>。

== 浏览器工具纠纷 ==
=== 漏洞纠纷 ===
2012年8月1日，有用户反映360安全卫士“漏洞修复”功能中编号为KB360018的“高危漏洞补丁”，并非由[[微软|微软]]公司发布。此补丁会将用户电脑内的[[Internet_Explorer_6|Internet Explorer 6]]浏览器升级为[[Internet_Explorer_8|Internet Explorer 8]]浏览器并可选安装'''360安全浏览器'''。<ref>{{cite web|url=http://news.mydrivers.com/1/236/236409.htm | title=360假冒发布系统补丁 微软官方或将介入调查 |publisher=驱动之家 |date=2012-08-01 |accessdate=2012-08-01| language = zh-hans}}</ref><ref>http://weibo.com/1709486153/yvbq1e1Ks</ref>
360官方也做出了正式回应，承认KB360018确实并非微软官方补丁，而是由360自行发布的“伪补丁”，初衷是“提供的IE6内核升级方案”。<ref>{{cite web|url=http://news.mydrivers.com/1/236/236587.htm | title=360紧急下架假冒补丁|publisher=驱动之家 |date=2012-08-03 |accessdate=2012-08-03| language = zh-hans}}</ref>

=== 安全性纠纷 ===
2014年10月20日，根据GreatFire.org组织的一条报道显示，中国防火墙[[防火长城|GFW]]正在通过一张假的安全证书，对苹果的[[iCloud|iCloud]]发起[[中间人攻击|中间人攻击]]。使用[[Google_Chrome|Google Chrome]]或[[Firefox|火狐浏览器]]时，用户会被警告证书的安全性，但360浏览器则没有防范。<ref>[https://zh.greatfire.org/blog/2014/oct/china-collecting-apple-icloud-data-attack-coincides-launch-new-iphone 苹果iCloud在中国遭中间人攻击]{{zh-cn}}</ref>

360官方回应称该报道不实，并表示360浏览器可以识别假证书，并在地址栏和信息栏上显示警告信息<ref>{{cite web|url=https://cabforum.org/pipermail/public/2014-October/004275.html|title=[cabfpub] 答复: China MITMing icloud.com|author=高寒蕊|date=2014年10月21日}}{{en}}</ref>。但又被指出360浏览器虽然会显示警告，但仍然打开页面，导致[[Cookie|Cookie]]泄露<ref>{{cite web|url=https://cabforum.org/pipermail/public/2014-October/004278.html|author=Gervase Markham|date=2014年10月22日|title=[cabfpub] 答复: China MITMing icloud.com}}{{en}}</ref>。

=== 隱私、資安疑慮 ===

==== 第一次[[App_Store_(iOS)|App Store]]下架 ====
2012年2月6日下午，奇虎360公司所有的iOS应用被苹果从[[App_Store_(iOS)|App Store]]上下架，苹果公司并未透露下架原因。<ref name="app-store-off-shelf-2012">{{cite web |url=http://www.chinanews.com/it/2012/02-07/3650345.shtml |title=苹果封杀360：360所有产品被App Store下架 |date=2012年2月7日 |website=中国新闻网 |accessdate=2012年2月7日}}</ref> 

2012年2月7日，奇虎360公司通过[[新浪微博|新浪微博]]声明：{{cquote|我们已获得苹果回复：360产品无需做任何修改，将在未来48小时到72小时内重新上架苹果APP商店。此次下架原因，系苹果方面发现部分产品被刷票，出现异常的用户好评和差评，苹果按惯例进行调查。由于360所有无线产品均在同一个苹果账号下，故导致了全线产品下架。}}<ref>{{cite web |url=http://tech.sina.com.cn/i/2012-02-07/10366691853.shtml |title=360称遭苹果下架因部分产品被刷票 |date=2012年2月7日 |website=新浪网 |accessdate=2012年2月7日}}</ref><ref>{{Cite web |url = http://weibo.com/1645903643/y4m3Zgdtl?type=comment |title = 我们已获得苹果回复... |author = 360安全卫士 |date = 2012-02-07 }}</ref>

2012年2月8日上午，360全线产品全部在[[App_Store_(iOS)|App Store]]重新上架。<ref>{{cite web |url=http://news.xinhuanet.com/it/2012-02/09/c_122676480.htm |title=360产品重新上架苹果App商店 |date=2012年2月9日 |website=新华网 |accessdate=2013年9月29日}}</ref>

==== 第二次[[App_Store_(iOS)|App Store]]下架 ====
2013年1月25日晚间，360公司在苹果[[App_Store_(iOS)|App Store]]上的大部分应用再次被下架，仅剩360云盘仍可下载。<ref>{{cite web |url=http://news.xinhuanet.com/info/2013-01/27/c_132130596.htm |title=时隔一年 360应用再遭苹果下架 |date=2013年1月27日 |website=新华网 |accessdate=2013年9月29日}}</ref>

2014年3月5日，360产品[[App_Store_(iOS)|App Store]]已重新上架。

=== 小米应用商店 ===
*2013年4月28日，360手机卫士被小米公司从MIUI应用商店下架，原因是在小米官方并不知情的情况下，该应用在提示用户修改权限的界面中，标注了“MIUI温馨提示”等字样，可能对用户产生误导。<ref name="xiaomi-off-shelf-1">{{cite web |url=http://tech.sina.com.cn/i/2013-04-29/15328293552.shtml |title=小米MIUI应用商店下架360安全卫士 |date=2013年4月29日 |website=新浪科技 |accessdate=2013年9月29日}}</ref>

*2013年9月27日，360全线产品被小米公司从小米应用商店下架，原因是360手机助手在没有给出任何理由的情况下，建议用户卸载小米应用商店、百度地图等其他公司的手机应用。<ref name="xiaomi-off-shelf-2">{{cite web |url=http://tech.sina.com.cn/i/2013-09-27/11318777298.shtml |title=小米应用商店全线下架360产品 |date=2013年9月27日 |website=新浪科技 |accessdate=2013年9月29日}}</ref>

*2016年1月8日，360全线产品被小米公司从小米应用商店下架，原因是部分小米用户被360安全卫士电脑版通过弹窗等形式恶意诱骗安装一款名为“雷电OS”的软件，在不知情情况下被篡改MIUI系统签名，导致小米手机无法正常升级系统，并通过篡改手机recovery阻止用户刷机卸载，甚至系统崩溃无法启动。<ref name="xiaomi-off-shelf-3">{{cite web |url=http://www.cnbeta.com/articles/464915.htm |title=小米发声明谴责“雷电OS”软件篡改MIUI系统签名 |date=2016年1月8日 |website=cnBeta.COM |accessdate=2016年1月9日}}</ref>

=== 其他应用商店 ===

== 水滴直播疑似侵犯隱私 ==
2017年4月，旗下的[[水滴直播|水滴直播]]平台上出现中国全国各地多家学校的课堂实时画面，涵盖幼儿园到高中。对此，家长的态度不一，一方认为直播平台“见证了孩子的点滴”，另一方则担心安全隐患，学生则坚决反对，觉得隐私受到侵犯。部分学校认为公开画面不妥，已经将直播关闭<ref>{{cite web |url=https://finance.sina.cn/2017-04-26/detail-ifyepsch3418151.d.html |title=水滴直播平台直播课堂宿舍 专家：直播不宜社会公开 |date=2017年4月26日 |publisher=三湘都市报 |accessdate=2017年12月14日}}</ref>。之后，水滴直播回应称摄像机默认关闭“水滴直播”功能，是否开启完全由用户自己决定。然而，《[[南方都市报|南方都市报]]》进一步报道水滴直播的场所不仅涉及学校，还对准了游泳池、盲人按摩馆、酒店大堂、内衣店，一些短租公寓室内等较为私密的场所也被直播<ref>{{cite web |url=https://xw.qq.com/tech/20170429026528/TEC2017042902652800 |title=连内衣店、按摩馆都被大量直播 画面让人心惊|date=2017年4月29日 |publisher=三湘都市报 |accessdate=2017年12月14日}}</ref>。后来在同年12月13日，90后女生陈菲菲通过微信公众号“菲言菲语”发表一篇题为《一位92年女生致周鸿祎：别再盯着我们看了》的文章，通过亲身素食店、餐厅、小吃店、网吧等多个安装了“小水滴”摄像头的店家，再度指控水滴直播侵犯隐私。这一次，董事长[[周鸿祎|周鸿祎]]在其个人微信公众号回应称，陈菲菲的文章误导读者，混淆视听，有“黑公关”之嫌。他还表示“这回被黑，肯定是又断了别人的财路了”<ref>{{cite web |author1=周卫 |author2=蔡晓颖|url=http://www.bbc.com/zhongwen/simp/chinese-news-42336636 |title=水滴直播：监控视频大家看 “被直播”凭什么|date=2017年12月13日 |publisher=BBC中文网|accessdate=2017年12月14日}}</ref>。陈菲菲则澄清自己并非“黑公关”，“从始至终没有收一分钱”，“也没有花一分钱用来推广”，表示会采取法律行动<ref>{{cite web |url=https://xw.qq.com/tech/20171213030527/TEC2017121303052700 |title=【一线】92年女生再度回应周鸿祎：没收一分钱，更没有玛莎拉蒂|date=2017年12月13日 |publisher=腾讯科技 |accessdate=2017年12月14日}}</ref>。2017年12月20日，360宣布永久关闭水滴直播<ref>{{cite web |url=https://xw.qq.com/tech/20171213030527/TEC2017121303052700 |title=360宣布永久关闭水滴直播|date=2017年12月20日 |publisher=腾讯财经 |accessdate=2017年12月20日}}</ref>。

==视频应用涉嫌大范围盗取视频==
2018年2月20日凌晨，有[[bilibili|bilibili]]的up主发现视频APP“[[快视频|快视频]]”在未经up主允许的情况下，大量盗取bilibili的视频，连up主的ID和视频下的评论也一并不经修改直接盗取，B站绑定的手机号码和密码能直接登录<ref name=ctoutiao>{{cite web|url=http://www.ctoutiao.com/517593.html|title=360快视频深陷盗用视频疑云，B站绑定的手机号码和密码疑似能直接登录}}</ref>。同日，bilibili通过认证号“up主首席执事”发布《关于某视频App盗传稿件事件的公告》呼吁维权<ref>{{cite web|author=up主首席执事|url=http://www.bilibili.com/read/cv232560|title=关于某视频App盗传稿件事件的公告|publisher=bilibili|date=2018-02-20}}</ref>。

12时55分，快视频回应称，快视频不存在数据库脱库、获取用户个人隐私的任何行为。部分博主传播的恶意诽谤、造谣传谣的言论严重损害快视频的声誉，快视频已对造谣行为进行报警处理，并将追究相应法律责任<ref>[http://tech.ifeng.com/a/20180220/44882907_0.shtml 将B站脱库？快视频回应：严重造谣已报警]</ref>。

14时17分，快视频回应称，经过排查发现部分快视频账号确实存在未经原创作者授权，私自搬用其内容，甚至冒用其头像及ID的行为，但否认存在盗取用户的帐号密码的行为<ref name=ctoutiao></ref>。甚至大量窃取[[新浪微博|新浪微博]]的用户信息以及其秒拍视频资源。

0时17分，bilibili发表了《关于用户反馈B站账号可以直接登录360快视频的调查结果》将“使用快视频登录输入B站账号密码后，B站账号会被异地登陆”和“用B站账号密码能直接登录360快视频”的流言证伪。官方在知乎上声称：“至今为止，我们没有发现B站的用户账号信息被泄漏的证据。账号安全是我们工作的重中之重，我们还在继续排查各种可能性。”<ref>[http://finance.ifeng.com/a/20180220/15990770_0.shtml 网传B站账号密码可直接登录360快视频 B站回应]，凤凰网</ref>

2月21日，bilibili正式向快视频发送律师函，要求快视频停止侵权行为，并向所有bilibiliUP主及bilibili公开赔礼道歉<ref>[http://news.zol.com.cn/678/6787835.html B站正式向360快视频发送律师函 要求停止侵权]</ref>。
