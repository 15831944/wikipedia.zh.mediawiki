{{字节单位}}
'''吉[[字节|字节]]'''（{{lang-en|Gigabyte}}，縮寫為'''GB'''），又稱-{zh-cn:'''吉咖字节'''、'''京字节'''或'''戟'''; zh-tw:'''十億位元組''';}-，是一种[[十進位|十進位]]的資訊计量[[单位|单位]]。吉位元組（Gigabyte）常容易和[[二進位|二進位]]的資訊計量單位[[Gibibyte|Gibibyte]]混淆。常使用在标示[[硬盘|硬盘]]、[[記憶體|記憶體]]等具有較大容量的储存媒介之储存容量。

== 与其他储存单位的换算 ==
* 1GB = 1,000（10<sup>3</sup>）[[MB|MB]]
* 1GB = 1,000,000（10<sup>6</sup>）[[KB|KB]]
* 1GB = 1,000,000,000（10<sup>9</sup>）[[byte|B]]

不過由於混淆已經普遍化，所以Gigabyte往往可以是指[[Gibibyte|Gibibyte]]。<br><br>
Gibibyte的换算是：
* 1GiB = 1,024（2<sup>10</sup>）[[MiB|MiB]] 
* 1GiB = 1,048,576（2<sup>20</sup>）[[KiB|KiB]]
* 1GiB = 1,073,741,824（2<sup>30</sup>）[[B|B]]

然而這兩者换算方式的不同，往往造成混淆。例如，一般的硬盘生产厂商采用1,000,000,000（10<sup>9</sup>）字节 = 1GB的进制，而[[Microsoft_Windows|Microsoft Windows]][[操作系统|操作系统]]中的换算公式则是1 GB = 2<sup>30</sup> = 1,073,741,824 字节（因為Windows對''GB''這個資訊計量單位的誤用，因此在Windows中顯示的"1GB"，其實應是指"1GiB"，但Windows卻顯示為"1GB"，而常造成誤解。誤用會普遍化的一大因素，是因為Windows的作業系統佔有率高），由於两种换算方法的不同，使容量在計算上相差了7.3%，所以常有Windows系统报告的容量比硬盘标示的容量還要小的情况发生。但在[[苹果公司|苹果公司]]的[[OS_X|OS X]]作業系統中，对于[[儲存裝置|儲存裝置]]的容量计算方式与硬盘厂商一致，均为1GB = 1,000,000,000（10<sup>9</sup>）字节的十进制，避免了计算和使用上的麻烦。
