'''向下兼容'''（downward compatibility），又称'''向后-{}-兼容'''（backward compatibility）、'''回溯-{}-相容'''，在[[计算机|计算机]]中指在一个[[程序|程序]]、[[库_(计算机)|库]]或[[硬體|硬體]]更新到较新[[版本|版本]]后，用旧版本程序创建的文档或系统仍能被正常操作或使用（包括输入数据）、在旧版本库的基础上开发的程序仍能正常[[编译|编译]]运行，或較舊版的硬體仍可在新版使用的情况。<!-- 2015秋季，家用遊戲機[[Xbox_One|Xbox One]]開放使用向下相容遊玩[[Xbox_360|Xbox 360]]，支援的遊戲日漸增加。 -->

==軟體==
在[[文件系统|文件系统]]中，[[ext4|ext4]]文件系统的设计就是向后兼容的，ext3的文件系统可以被当作ext4文件系统挂载。

通常更新一个软件时，应该为向下兼容性做出一定的考虑，这往往能给用户带来方便并更好地留住用户。[[Microsoft|Microsoft]] 特別強調維持軟件的向下兼容性。為了實現此一目標，有時微軟甚至不惜支持使用了非官方乃至误用的 API 的軟件。

但情况并不总是这样，有时为了考虑向下兼容会带来一些累赘，甚至增加開發的成本，尤其是进行过较多[[升级|升级]]后。[[Python|Python]] 3.0 便是放弃向下兼容的一个例子<ref name="py3k_whatsnew">{{cite web|language=en |publisher=Python Software Foundation |title=What's New in Python 3.0--Python v3.0b3 documentation |url=http://docs.python.org/dev/3.0/whatsnew/3.0.html |author=Guido van Rossum |date= |accessdate=2008-08-30 |deadurl=yes |archiveurl=https://web.archive.org/web/20081004052720/http://docs.python.org/dev/3.0/whatsnew/3.0.html |archivedate=2008-10-04 }}</ref><ref>为了照顾现有程序，[[Python|Python]] 提供了一个 [[Python|Python]] 2.6 -> [[Python|Python]] 3.0 的代码转换工具. {{cite web |url=http://svn.python.org/view/sandbox/trunk/2to3/ |title=存档副本 |accessdate=2008-08-30 |deadurl=yes |archiveurl=https://archive.is/20120729222226/http://svn.python.org/view/sandbox/trunk/2to3/ |archivedate=2012-07-29 }}</ref>。

如果一个软件的旧的版本保存的文档不能被新版本读取，即使厂商提供了额外工具对旧文档进行不可逆转的转换，这也不能称作向下兼容。这可能是软件厂商的一种市场战略，强迫用户购买升级软件，随着[[升级|升级]]用户逐渐增加，逐渐地使得旧软件能读取的文档越来越少。

==硬體==
以廣為使用的[[通用序列匯流排|通用序列匯流排]]（USB）為例，從1998年的1.1版到2015年時的3.1版，A型的插頭與插座均可跨版本互相使用，僅新版本所支援的較大傳輸速率與一些新特性無法為舊版所使用。

[[藍光光碟|藍光光碟機]]通常可向下相容讀取[[DVD|DVD]]、[[CD|CD]]。

此外還有微軟公司推出的[[Xbox_One|Xbox One]]遊戲機對[[Xbox_360|Xbox 360]]，和[[任天堂|任天堂]]推出的[[Wii_U|Wii U]]對[[Wii|Wii]]的旧版本游戏在新版本硬件中提供支持。但是[[索尼電腦娛樂|索尼電腦娛樂]]推出的[[PS4|PS4]]遊戲機則不向下相容[[PS3|PS3]]的遊戲。
