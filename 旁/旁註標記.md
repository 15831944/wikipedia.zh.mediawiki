'''旁註標記'''（{{lang|en|ruby character}}），或稱'''注音標示'''、'''加-{}-注音'''、'''Ruby字元'''、'''ruby'''或'''rubi'''，是一種[[表意文字|表意文字]]的[[音標|音標]]印刷方式，廣泛地運用於[[日文|日文]]及[[中文|中文]]。一般這些字是放於表意文字的上方或右邊，作為文字的拼音或註解。

「Ruby」或「rubi」是[[日本|日本]]印刷業的用語，非[[術語|術語]]的稱呼則是[[振假名|振假名]]（{{jpn|j='''振り仮名'''|rm=Furigana}}）。在台灣，由於標註的多為[[注音符號|注音符號]]，故多稱為「注音標示」、「加-{}-注音」或「加-{}-註注-{}-音」。如Microsoft Word繁體中文版的Ruby功能就譯為「注音標示」。在中国大陆，用于标注的多为[[汉语拼音|汉语拼音]]，多见于儿童刊物，一般称做“给汉字注音”或简称“注音”。

在中文裡，通常使用[[漢語拼音|漢語拼音]]或[[注音符號|注音符號]]做旁註標記，但一般不在報章等一般-{文件}-或圖書使用標音旁註標記，通常只用於小學的教科書或辭典，或兒童圖書等。由於中國大陆通常中文只用橫寫，所以一般只見上方的旁註標記。台灣橫直寫均用，一般兒童讀物多直寫，注音也標示在右方為主。例如台灣每日發行的《[[國語日報|國語日報]]》，所有的報導文章都使用注音旁註標記。

==用途==
在书写日文和中文时，有些单字的发音经常被标注在字的上方或右方。这个动作叫做[[標音|注音]]；它们不止用于印刷品，而且用于手写中。这可能由于以下几种原因：

* 因为这个字是生僻字，很多人不知道发音
* 因为这个字有多种发音，且不能从上下文中推测
* 因为这篇文章是给儿童（或外语学习者）看的，他们对发音不太了解
* 在日语的笑话和漫画中用于强调，通常是双关语，中國大陸和台灣亦有類似的[[注音文|注音文]]。 
* 在日语中，使用一个不熟悉的外来语或俚语，注音是为了表示它的含义，而且这些字和意思没有关系，通常使用假名

'''Ruby'''原指5.5級的字体格式，在美国又称'''Agate'''，它的大小正好适合注音。现在在日文中，这词不再表示'''字体的名称'''，而是表示名词形式的'''[[標音|標音]]'''。当它被翻译回英文时，它被称为'''rubi'''（由假名译来，日文：{{lang|ja|ルビ}}）。但是Ruby的拼法还是比较普遍。

除了字母以外，一些假名也会用Ruby来标注。

==瀏覽器的旁註標記功能==
2011年，W3C發佈[[XHTML|XHTML]]旁註標記功能<ref name="w3c-spec">
{{cite web
|author=Marcin Sawicki, Michel Suignard, Masayasu Ishikawa, Martin Dürst, Tex Texin
|title=Ruby Annotation
|date=2001-05-31
|work=[[W3C_Recommendation|W3C Recommendation]]
|publisher=[[World_Wide_Web_Consortium|World Wide Web Consortium]]
|accessdate=2007-02-14
|url=http://www.w3.org/TR/ruby
}}
</ref>，這個功能不是HTML 4.01的一部分或XHTML 1.0規格（XHTML-1.0-Strict、XHTML-1.0-Transitional，以及XHTML-1.0-Frameset）中的任何一部分，而是[[XHTML|XHTML]] 1.1規格中的其中一部分，并且作為HTML5規格的核心部分由W3C完成制定。<ref>{{cite web|title=HTML5|url=http://www.w3.org/TR/html5/text-level-semantics.html#the-ruby-element|accessdate=2014-11-12|deadurl=yes|archiveurl=https://web.archive.org/web/20150801133040/http://www.w3.org/TR/html5/text-level-semantics.html#the-ruby-element|archivedate=2015-08-01}}</ref>

Ruby標籤得到微軟[[Internet_Explorer|Internet Explorer]]（5.0及以上）和[[Google_Chrome|Google Chrome]]以及Mac的[[Safari|Safari]]的部分支援，但[[Konqueror|Konqueror]]和[[Opera|Opera]]（15.0版本之前）都不支援Ruby標籤。WebKit nightly版在2010年1月添加Ruby标签支持<ref name="webkit_ruby">{{cite web |url=http://webkit.org/blog/948/ruby-rendering-in-webkit/ |title=Ruby Rendering in WebKit |author=Roland Steiner |date=2010-01-20 |work=Surfin’ Safari |publisher=[[WebKit|WebKit]] project |accessdate=2010-01-21}}</ref>，[[Safari|Safari]]在5.0.6版加入该功能<!-- may have been introduced in one of the other 5.0.x updates --><ref>[http://html5doctor.com/ruby-rt-rp-element/ The ruby element] - HTML5 Doctor</ref>，Mozilla Firefox 38也提供支持。<ref name="fx38"></ref><ref>{{cite web |url=https://www.mozilla.org/en-US/firefox/38.0/releasenotes/ |title=Mozilla Firefox 38.0 Release Notes |publisher=Mozilla.org |accessdate=2015-04-10}}</ref>

在Ruby标签未受广泛支持的情况下，可以通过使用[[CSS|CSS]]来实现。<ref name="ruby_stylesheet_hack">[http://web.nickshanks.com/stylesheets/ruby.css CSS Ruby支持] {{webarchive|url=https://web.archive.org/web/20070228150240/http://web.nickshanks.com/stylesheets/ruby.css |date=2007-02-28 }}——适用于所有浏览器</ref>

Ruby标签也可以添加到允许自定义扩展的浏览器当中。

Ruby标签经结构化以达到向后兼容，以在主要文本后面包含在括号中的旁注字符组成，会在不支持ruby标签的浏览器里面显现出来。

W3C还为[[CSS|CSS]] level 2制定ruby模块，能允许额外自动分组和自动略去与与注释对象相同的假名。<ref name="css2_ruby">{{cite web
|author = Elika J. Etemad, Koji Ishii|title = CSS Ruby Layout Module Level 1|date = 2015-04-16|work = W3C Editor’s Draft|publisher = [[World_Wide_Web_Consortium|World Wide Web Consortium]]|accessdate = 2015-05-05|url = http://dev.w3.org/csswg/css-ruby-1/}}</ref>当前仅有Firefox 38提供支持。<ref name="fx38">{{cite web|url=https://hacks.mozilla.org/2015/03/ruby-support-in-firefox-developer-edition-38/|title=Ruby support in Firefox Developer Edition 38|date=2015-03-05|author=Xidorn Quan|publisher=Mozilla|accessdate=2015-05-05}}</ref>

==Unicode的旁註標記功能==
[[Unicode|Unicode]]和[[通用字符集|通用字符集]]通过行间注释（interlinear annotation）符号来实现旁注标记功能：
* <code>U+FFF9</code>—Interlinear annotation anchor—标记要注释的文字
* <code>U+FFFA</code>—Interlinear annotation separator—注释文字的开头
* <code>U+FFFB</code>—Interlinear annotation terminator—注释文字的结尾
用法如下所示：
* -{<code>U+FFF9</code>東<code>U+FFFA</code>とう<code>U+FFFB</code><code>U+FFF9</code>京<code>U+FFFA</code>きょう<code>U+FFFB</code>}-
* <code>U+FFF9</code>北<code>U+FFFA</code>ㄅㄟˇ<code>U+FFFB</code><code>U+FFF9</code>京<code>U+FFFA</code>ㄐ丨ㄥ<code>U+FFFB</code>
结果为：
* -{￹東￺とう￻￹京￺きょう￻}-
* ￹北￺ㄅㄟˇ￻￹京￺ㄐ丨ㄥ￻

只有少数程序支持这个功能。Unicode技术报告#20<ref name="unicode20">{{cite web
|    author = Martin Dürst, Asmus Freytag
|     title = Unicode in XML and other Markup Languages
|      date = 2007-05-16
| publisher = [[W3C|W3C]]和[[統一碼聯盟|統一碼聯盟]]
|       url = http://www.unicode.org/unicode/reports/tr20/
}}</ref>说明这些字符不能用于标记语言和应用程序，建议在适当的地方代之以旁注标记。
