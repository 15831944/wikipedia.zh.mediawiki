在[[電腦網路|電腦網路]]中，'''黑洞'''代指静默丢弃进出流量（亦或称为“丢包”）的网路地点，且不通知源头数据未到达指定的接受者。

当检查[[网路拓扑|网路拓扑]]时这些黑洞不可见，且只能以通过监测遗失的流量来发现它们。故名“[[黑洞|黑洞]]”。

== 死地址 ==
最常见的的黑洞形式是一个IP地址所在的特定主机未在运行或IP地址未被指定。

即使[[TCP/IP协议族|TCP/IP]]协议提供了几种通过[[ICMP|ICMP]]的通讯方式使得传输故障的信息可以传回发送者，但这些前往死地址的流量通常直接被丢弃。

注意死地址只相对那些无连接传输模式和不可靠的传输协议来说不可检测（例如[[用户数据报协议|UDP]]）. 而连接导向传输或可靠的传输协议（如[[传输控制协议|TCP]]和[[RUDP|RUDP]]） 会无法连接到死地址或者无法收到预期的确认。

== 防火墙与隐形端口 ==
大多数[[防火墙|防火墙]]（家用路由器）可以被设置为将发给不被允许的主机或端口的[[数据包|包]]静默丢弃，从而导致网络内小范围或大范围的“黑洞”。

不响应[[ICMP|ICMP]]回应（Ping）请求的个人防火墙被某些供应商<ref>Apple Inc., [http://support.apple.com/kb/HT1810 botXhacker" "About the Application Firewall"]</ref>指定为处于“隐身模式”。

尽管如此，大多数网络中的拥有防火墙的主机可轻松区分无效或不可达的IP地址。如果遇到后者，[[路由器|路由器]]通常会回复一个ICMP网络rsp的应答。用于家用和办公用路由器中的[[网络地址转换|NAT]]，通常是一种更有效的方式去模糊内部网络的布局。

== 黑洞过滤 ==
黑洞过滤所指的是在路由层次丢弃数据包，通常使用[[路由协议|路由协议]]来补充几个路由器上的过滤，时常可以快速动态响应[[拒绝服务攻击|拒绝服务攻击]]。

== 域名服务器黑名单（互联网流量管理） ==
域名服务器黑名单（DNSBL），亦或称为实时阻止列表（RBL）是指由域名信息文件提供给[[域名解析服务|域名解析服务]]所产生的[[IP地址|IP地址]]列表。域名服务器黑名单通常可以用于公布[[濫發電子訊息|滥发电子消息]]的计算机或网络，大多数的[[Mail_transport_agent|电子邮件服务器]]可被设置为拒收或标记黑名单中的计算机或网络所发出的消息。
