'''通用缺陷列表'''（Common Weakness Enumeration, CWE）是一个对软件脆弱性和易受攻击性的一个分类系统。它由一个的开源社区工程维持，该社区有理解软件中的瑕疵并创作对自动化工具去识别、修复并防止这些瑕疵的愿景<ref>{{cite web |author1=Miter Corp. |title=CWE-About CWE |url=http://cwe.mitre.org/about/index.html}}</ref>。该工程由下属于{{le|Mitre集团公司|Mitre Corporation}}的{{le|美国联邦基金研究发展中心-国家网络安全小组|National Cybersecurity FFRDC}}资助，并得到了来自{{le|美国国家网络应急中心|United States Computer Emergency Readiness Team}}和[[美国国土安全部|美国国土安全部]]的{{le|国家网络安全办公室}}的支持。<ref>{{cite web |author1=nist.gov |title=NATIONAL VULNERABILITY DATABASE |url=https://nvd.nist.gov/vuln/categories}}</ref>通用缺陷列表标准3.2版本在2019年1月发布。<ref>{{cite web |author1=The Mitre Corp. |title=CWE Version 3.2 Now Available |url=https://cwe.mitre.org/news/index.html#jan032019_CWE_Version_3.2_Now_Available}}</ref>通用缺陷列表有超过600個的分类，包括[[缓冲溢出|缓冲溢出]]，路径索引遍历树错误，[[竞争状态条件|竞争状态条件]]，[[跨站脚本|跨站脚本]]，硬编码密码以及不安全[[随机数|随机数]]等。<ref>{{cite web |author1=nist.gov |title=Common Weakness Enumeration (CWE) Class |url=https://samate.nist.gov/BF/Enlightenment/CWE.html}}</ref>

== 通用缺陷列表兼容性 ==
通用缺陷列表兼容性项目允许一个服务或产品被审核并被官方地注册为“通用缺陷列表兼容的”和“通用缺陷列表有效的”。该项目协助组织选择正确的软件工具并了解到可能的脆弱性和它的影响。为了获得通用缺陷列表兼容的状态，一个产品或者一个服务必须满足下列6项中的4项要求：

{| class="wikitable"
|-
| 通用缺陷列表-可搜索的 || 用户可用通用缺陷列表标识符查找安全元素
|-
| 通用缺陷列表-输出 || 呈现给用户的安全元素包括，或允许用户通过通用缺陷列表标识符获得
|-
| 映射准确性 || 安全元素能准确的链接到合适的通用缺陷列表标识符
|-
| 通用缺陷列表文档 || 描述通用缺陷列表的功能性文档和通用缺陷列表兼容性，以及通用缺陷列表相关的功能是怎样使用的
|-
| 通用缺陷列表覆盖 || 对于通用缺陷列表兼容性和有效性，功能性文档显示地列出了通用缺陷标识号，使得
|-
| 通用缺陷列表测试结果 || 对于通用缺陷列表有效性，来自于功能性的测试结果展现了软件评估的结果，并会被展示在通用缺陷列表网站上
|}
截止2018年1月，一共有48家组织开发并维护达到通用缺陷列表兼容性的产品和服务。<ref>{{cite web |author1=CWE Community |title=CWE-Compatible Products and Services |url=https://cwe.mitre.org/compatible/compatible.html}}</ref>
