===多方面的近似值===
在古代，人們使用60進制來計算。在60進制中，{{pi}}能被準確至小數點後八位（十進制），而這數字是3:8:29:44{{sub|60}}，即是：

:<math> 3 + \frac{8}{60} + \frac{29}{60^2} + \frac{44}{60^3} = 3.14159\ 259^+</math>

（下一個[[60進制|60進制]]的數位為0）

除此之外，{{pi}}還能以以下方式表示：

*準確至3位：
:: <math>\sqrt{2} + \sqrt{3} = 3.146^+</math>

*準確至4位：
::<math>\sqrt{7+\sqrt{6+\sqrt{5}}} = 3.1416^+</math><ref>[http://www.mschneider.cc/papers/pi.pdf A nested radical approximation for pi] {{webarchive|url=https://web.archive.org/web/20110706215615/http://www.mschneider.cc/papers/pi.pdf |date=2011-07-06 }}</ref>

*準確至4位：
::<math>\sqrt[3]{31} = 3.1413^+</math><ref>{{Cite document|first=Martin|last=Gardner|authorlink=Martin Gardner|title=New Mathematical Diversions|publisher=Mathematical Association of America|year=1995|page=92|postscript=<!-- Bot inserted parameter. *[[拉馬努金|拉馬努金]]的近似值，準確至4位：
::<math>\frac{9}{5}+\sqrt{\frac{9}{5}} = 3.1416^+</math>

===圓形的面積===
可以通过[[蒙特卡洛方法|蒙特卡洛方法]]来计算圆周率<math>\pi</math>。

以原点（0, 0）为圆心，画一个半径为<math>r</math>的圆。然后以原点为中心，画一个边长为<math>2r</math>的正方形。圆和正方形内切。

圆的面积为<math>A_1=\pi*r^2</math>，正方形的面积为<math>A_2=(2r)^2=4r^2</math>。

于是有，<math>A_1/A_2=\pi/4</math>。

通过生成0到r之间随机数作为一个点的横纵坐标，所有点均落在正方形内。

通过统计圆内的点数<math>N_{inside}</math>与总点数<math>N_{total}</math>，<math>\pi=4*A_1/A_2=4*N_{inside}/N_{total}</math>。

当随时点的数目增加时，所得结果会越接近于圆周率。

但是该方法也有不足之处。具体可参考[[蒙特卡洛方法|蒙特卡洛方法]]。

===連分數===
{{pi}}的[[連分數|連分數]]表示式是[3; 7, 15, 1, 292, 1, 1, ...]。這連分數沒有任何模式。{{pi}}有很多用一條簡單的規矩然製成的[[廣義連分數|廣義連分數]]

:<math>
\pi= {3 + \cfrac{1^2}{6 + \cfrac{3^2}{6 + \cfrac{5^2}{6 + \ddots\,}}}}\!
</math>

:<math>
\pi = \cfrac{4}{1 + \cfrac{1^2}{3 + \cfrac{2^2}{5 + \cfrac{3^2}{7 + \ddots}}}}\!
</math>

（其他連分數能在[http://functions.wolfram.com/Constants/Pi/10/ 這裡]查看。）

==計算任意數位的方法==
在1995年，[[西蒙·普勞夫|西蒙·普勞夫]]發現了[[贝利－波尔温－普劳夫公式|贝利－波尔温－普劳夫公式]]。這公式能在[[16進制|16進制]]中計算{{math|pi}}的任意數位，而不用計算之前的數位。<ref>MathWorld: BBP Formula [http://mathworld.wolfram.com/BBPFormula.html Wolfram.com]</ref>

: <math>\pi=\sum_{n=0}^\infty \left(\frac{4}{8n+1}-\frac{2}{8n+4}-\frac{1}{8n+5}-\frac{1}{8n+6}\right)\left(\frac{1}{16}\right)^n\!</math>

在1996年，[[西蒙·普勞夫|西蒙·普勞夫]]發明了一個公式，能在[[大O符號|O]]({{math|n}}<sup>3</sup>log({{math|n}})<sup>3</sup>)的時間之內計算出{{math|pi}}在任意進制的第{{math|n}}個數位<ref>Simon Plouffe, ''On the computation of the n'th decimal digit of various transcendental numbers'', November 1996</ref>。在1997年，[[法布里斯·贝拉|法布里斯·贝拉]]發明了另一個公式，把計算所需時間縮短至[[大O符號|O]]({{math|n}}<sup>2</sup>)。他又發明了在2進制計算{{math|pi}}的公式。<ref>Bellard's Website:[http://bellard.org/pi/pi_n2/pi_n2.html Bellard.org]</ref>

: <math>\pi=\frac{1}{2^6}\sum_{n=0}^\infty \frac{(-1)^n}{2^{10n}} \left (-\frac{2^5}{4n+1}-\frac{1}{4n+3}+\frac{2^8}{10n+1}-\frac{2^6}{10n+3}-\frac{2^2}{10n+5}-\frac{2^2}{10n+7}+\frac{1}{10n+9}\right )\!</math>
==有效的方法==
在1961年，{{link-en|丹尼爾柄|Daniel Shanks}}和他的團隊在[[美國海軍研究實驗室|美國海軍研究實驗室]]計算了{{pi}}的前100,000數位。

他和他的團隊使用了兩個不同的幂級數來計算{{pi}}的數值。第一個幂級數中，任何錯誤都會造成一個比較高的數值；而另一個中，任何錯誤都會造成一個比較低的數值。所以如果兩個幂級數計算出同樣的數值，那個數值就肯定正確。美國海軍研究實驗室發放了{{pi}}的前100,000數位。

但是以上的兩個幂級數也要很長的時間才能計算出結果。相反地，{{link-en|約翰·梅欽|John Machin}}的公式與[[反正切|反正切]]的[[泰勒级数|泰勒级数]]一起使用則能很快地計算結果：

: <math>\frac{\pi}{4} = 4 \arctan\frac{1}{5} - \arctan\frac{1}{239}\!</math>

使用[[複數_(數學)|複數]]的[[極坐標系|極坐標系]]便能證實這公式，以以下的數學式開始：

:<math>(5+i)^4\cdot(239-i)=2^2 \cdot 13^4(1+i).\!</math>

這類的公式被稱為[[梅欽類公式|梅欽類公式]]。(注意，{ {{math|''x''}},{{math|''y''}}} = {239, 13<sup>2</sup>}是[[佩爾方程|佩爾方程]]「{{math|''x''}}<sup>2</sup>-2{{math|''y''}}<sup>2</sup> = -1」的其中一個解答。)

印度數學家[[斯里尼瓦瑟·拉马努金|斯里尼瓦瑟·拉马努金]]發現了{{pi}}的很多其他表示方式。他與[[戈弗雷·哈罗德·哈代|戈弗雷·哈罗德·哈代]]一起工作了很多年。

如果要計算{{pi}}小數點後很多位，計算者通常會使用[[高斯-勒让德算法|高斯-勒让德算法]]，{{link-en|波尔温公式|Borwein's algorithm}}，和1976年發明的[[薩拉明_-_布倫特公式|薩拉明 - 布倫特公式]]。

{{pi}}和<sup>1</sup>/<sub>{{pi}}</sub>的小數點後首十萬位能在[[古腾堡计划|古腾堡计划]]裡查閱(參見[[#外部連結|#外部連結]])。

在2002年12月，在[[東京大學|東京大學]]進修的[[金田康正|金田康正]]發放了{{pi}}小數點後1,241,100,000,000位的值，創造了新的世界記錄。他在2002年9月以六十四部[[日立|日立]]的[[超級電腦|超級電腦]]計算出這值。這些電腦有1TB的記憶體，而且能在每秒執行2兆次運算。上一個記錄(21億位)所使用的電腦每秒只能執行1兆次運算。金田康正使用了以下公式：

:<math> \frac{\pi}{4} = 12 \arctan\frac{1}{49} + 32 \arctan\frac{1}{57} - 5 \arctan\frac{1}{239} + 12 \arctan\frac{1}{110443}\!</math>
:K. Takano (1982).

: <math> \frac{\pi}{4} = 44 \arctan\frac{1}{57} + 7 \arctan\frac{1}{239} - 12 \arctan\frac{1}{682} + 24 \arctan\frac{1}{12943}\!</math>
:F. C. W. Störmer (1896).

這些近似值由於有太多數位，所以沒有實際用途，只是用來測試超級電腦。

在1997年，大衛·貝利（{{link-en|David H. Bailey|David H. Bailey}}）、{{link-en|皮特·波爾溫|Peter Borwein}}和[[西蒙·普勞夫|西蒙·普勞夫]]發佈了[[贝利－波尔温－普劳夫公式|一條新的公式]]來計算{{pi}}的值：

: <math>\pi = \sum_{k = 0}^{\infty} \frac{1}{16^k}
\left( \frac{4}{8k + 1} - \frac{2}{8k + 4} - \frac{1}{8k + 5} - \frac{1}{8k + 6}\right).\!</math>

這公式能在不知道前{{math|k}} - 1數位的值之下，在[[2進制|2進制]]或[[16進制|16進制]]中計算出{{pi}}的第{{math|k}}個數位的值。[http://crd.lbl.gov/~dhbailey/ 貝利的網頁]包含了計算方法，而且把方法以幾個[[程式語言|程式語言]]記下。{{link-en|PiHex|PiHex}}計算出{{pi}}小數點後一兆數位的值。

[[法布里斯·贝拉|法布里斯·贝拉]]推出了[[贝利－波尔温－普劳夫公式|贝利－波尔温－普劳夫公式]]的改良版——[[貝拉公式|貝拉公式]]：

:<math>\pi = \frac{1}{2^6} \sum_{n=0}^{\infty} \frac{{(-1)}^n}{2^{10n}} \left( - \frac{2^5}{4n+1} - \frac{1}{4n+3} + \frac{2^8}{10n+1} -\frac{2^6}{10n+3} - \frac{2^2}{10n+5} - \frac{2^2}{10n+7} + \frac{1}{10n+9} \right)\!</math>

還有其他計算{{pi}}的值的公式：

:<math>
\frac{\pi}{2}=\sum_{k=0}^\infty\frac{k!}{(2k+1)!!}=\sum_{k=0}^{\infty}\frac{2^k k!^2}{(2k+1)!} =1+\frac{1}{3}\left(1+\frac{2}{5}\left(1+\frac{3}{7}\left(1+\cdots\right)\right)\right)\!</math>
:[[艾萨克·牛顿|牛頓]]

:<math> \frac{1}{\pi} = \frac{2\sqrt{2}}{9801} \sum^\infty_{k=0} \frac{(4k)!(1103+26390k)}{(k!)^4 396^{4k}}\!</math>
:[[斯里尼瓦瑟·拉马努金|斯里尼瓦瑟·拉马努金]]

拉馬努金的公式收歛的速度異常地快，這公式後來在2000年演變成最快的公式：

:<math> \frac{1}{\pi} = 12 \sum^\infty_{k=0} \frac{(-1)^k (6k)! (13591409 + 545140134k)}{(3k)!(k!)^3 640320^{3k + 3/2}}\!</math>
:[[David_Chudnovsky|David Chudnovsky]]和[[Gregory_Chudnovsky|Gregory Chudnovsky]].

==計算圓周率近似值的軟件==
===General purpose===
大多数[[计算机代数系统|计算机代数系统]]可以计算出[[π|π]]和其他常见的[[数学常数|数学常数]]到任何所需的精度。

计算[[π|π]]的功能中还包括许多通用库任意[[高精度计算|精度算术运算]]，例如[[CLN|CLN]]和[[MPFR|MPFR]]。
